<task type="auto">
  <name>Task 2: Create customer settings API route</name>
  <files>
    src/app/api/account/settings/route.ts

  <action>
    Create `src/app/api/account/settings/route.ts` following the exact pattern from `src/app/api/account/profile/route.ts`:

    **GET handler:**
    1. `createClient()` from `@/lib/supabase/server`
    2. `supabase.auth.getUser()` — return 401 if no user
    3. Lazy row creation: `supabase.from("customer_settings").insert({ user_id: user.id }).select().maybeSingle()` — the DB's ON CONFLICT DO NOTHING handles existing rows
    4. Fetch row: `supabase.from("customer_settings").select("dietary_restrictions, delivery_instructions, notification_prefs, theme").eq("user_id", user.id).single()`
    5. Transform row to camelCase response:
       ```typescript
       function transformRow(row: CustomerSettingsRow) {
         return {
           dietaryRestrictions: (row.dietary_restrictions as string[]) ?? [],
           deliveryInstructions: row.delivery_instructions ?? "",
           notificationPrefs: (row.notification_prefs as NotificationPrefs) ?? { order_updates: true, marketing: true, reminders: true },
           theme: row.theme ?? "system",
         };
       }
       ```
    6. Return `NextResponse.json({ data: transformRow(data) })`

    **PATCH handler:**
    1. Auth check (same as GET)
    2. Parse body with `updateCustomerSettingsSchema.safeParse(body)` — return 400 on failure with `result.error.issues`
    3. Build snake_case update object from validated data (only include fields that are present)
    4. Upsert: `supabase.from("customer_settings").upsert({ user_id: user.id, ...updates, updated_at: new Date().toISOString() }, { onConflict: "user_id" })`
    5. Fetch fresh row after upsert and return transformed
    6. Wrap both handlers in try/catch with `logger.exception(error, { api: "account/settings" })`

    Import validation schema from `@/lib/validations/customer-settings`.
    Import `Json` type from `@/types/database` for JSONB casting.
    Error response format: `{ error: { code: string, message: string } }` matching profile route.
