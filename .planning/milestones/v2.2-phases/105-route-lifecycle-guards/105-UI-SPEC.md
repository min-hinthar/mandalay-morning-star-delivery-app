## Interaction Contract

### 1. Admin Route Status Dropdown (RouteHeader.tsx)

**Current behavior:** All 5 status options always selectable.

**Target behavior:** Only valid transitions are selectable. Invalid options shown disabled (50% opacity, no pointer events).

| Current Status | Enabled Options | Disabled Options |
|---------------|----------------|-----------------|
| planned | assigned | accepted, in_progress, completed |
| assigned | planned, accepted | in_progress, completed |
| accepted | planned, assigned, in_progress | completed |
| in_progress | completed | planned, assigned, accepted |
| completed | (none -- all disabled) | planned, assigned, accepted, in_progress |

**Implementation:** Import `getValidRouteTransitions()` from shared validation. Map over all statuses, set `disabled` prop on `SelectItem` for statuses not in valid transitions list. Current status remains the Select value (shown in trigger, not in options list since Radix excludes current).

**Feedback on rejection:** Backend returns 400 with `{ error, validTransitions }`. Existing `useStopMutations.ts` try/catch reverts optimistic update and shows error toast: "Failed to update route status". No changes to toast copy needed -- the guard prevents reaching this state from the UI.

### 2. Driver Start Route Button Visibility

**Current behavior (bug):** `ActiveRouteView.tsx:193` shows Start Route for `planned` OR `accepted` status.

**Target behavior:** Show Start Route button ONLY for `accepted` status.

**Files affected:**
- `ActiveRouteView.tsx:193` -- change `routeStatus === "planned" || routeStatus === "accepted"` to `routeStatus === "accepted"`
- `SimpleHome.tsx:111-113` -- already correct (only shows for `accepted`), verify no change needed

**No visual changes to the button itself.** Same green background, same Play icon, same 14px height, same animation.

### 3. Driver Accept/Decline Bar (No Changes)

Already correct:
- Shows for `assigned` status: Accept + Decline buttons
- Shows for `accepted` status: Decline button only
- Hidden for all other statuses

No modifications needed.
