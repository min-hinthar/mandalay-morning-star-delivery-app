    - Returns { handleReorder, isReordering }
    - On success: calls onSuccess callback
    - On error: calls onError callback with previous stops for revert
    - Fire immediately per drag-drop, no debounce

    useReassignDriver:
    - Accepts routeId, routeStatus, currentDriverName
    - reassignDriver(driverId: string) fires PATCH to /api/admin/routes/[routeId] with driverId
    - Returns { reassignDriver, isReassigning, showConfirmation, confirmReassign, cancelReassign, pendingDriverId }
    - For in_progress routes: sets showConfirmation=true and stores pendingDriverId, waits for confirmReassign()
    - For planned routes: fires immediately without confirmation
    - On success: shows success toast
    - On error: shows error toast
  </behavior>
  <action>
    **useReorderStops.ts** — Custom hook for optimistic stop reordering.
    - `'use client'` not needed (hook file, imported by client components)
    - Uses useState for isReordering flag
    - handleReorder maps StopDetail[] to { stopId, stopIndex } tuples
    - Fires fetch PATCH to `/api/admin/routes/${routeId}` with JSON body
    - Includes `forceOverride: true` when routeStatus === 'in_progress' (RESEARCH Pitfall 6)
    - On success: calls onSuccess()
    - On non-ok response: calls onError(previousStops) for revert, shows error toast via sonner
    - On catch: same revert behavior
    - No debounce per user decision

    **useReassignDriver.ts** — Custom hook for driver reassignment.
    - Uses useState for isReassigning, showConfirmation, pendingDriverId
    - reassignDriver(driverId): if in_progress, store pendingDriverId and set showConfirmation=true; if planned, fire immediately
    - confirmReassign(): fires PATCH to `/api/admin/routes/${routeId}` with { driverId: pendingDriverId }, clears confirmation state
    - cancelReassign(): clears confirmation state
    - On success: toast.success with driver reassigned message
    - On error: toast.error with failure message
