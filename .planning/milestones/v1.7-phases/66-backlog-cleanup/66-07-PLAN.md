**1. Console.log cleanup:**
- Scan all `src/` files for console statements:
  - `grep -rn "console\.log\b" src/ --include="*.ts" --include="*.tsx"` (bare console.log)
  - `grep -rn "console\.debug\b" src/ --include="*.ts" --include="*.tsx"` (debug)
  - `grep -rn "console\.warn\b" src/ --include="*.ts" --include="*.tsx"` (warn)
  - `grep -rn "console\.error\b" src/ --include="*.ts" --include="*.tsx"` (error)
- Removal rules (per CONTEXT.md discretion):
  - **KEEP:** `console.debug("[cart]"`, `console.warn("[cart]"` (prefixed = intentional debug)
  - **KEEP:** `console.error` in catch blocks (legitimate error logging)
  - **KEEP:** `console.warn` with development-only guard (`if (process.env.NODE_ENV === "development")`)
  - **REMOVE:** Bare `console.log(...)` without prefix or dev guard
  - **REMOVE:** `console.log` that are clearly debugging artifacts (logging variable names, "here", "test", etc.)
- Remove identified bare console.log statements

**2. CSS dead code scan:**

- Scan for Tailwind custom CSS classes defined in `globals.css` or `tokens.css` that have zero usage in `src/`
- For each custom class/token:
  - `grep -r "classname" src/ --include="*.tsx" --include="*.ts"` to check usage
  - Be careful with template literal usage: `text-${color}` patterns mean the token is used dynamically
- Only remove tokens with ZERO references (exact match and template match)
- **Conservative approach:** When uncertain, keep the token

**3. Unused TypeScript types scan:**

- Check for types/interfaces with zero import references:
  - Focus on `src/types/*.ts` files
  - For each exported type: `grep -r "TypeName" src/ --include="*.ts" --include="*.tsx" | grep -v "^src/types/"` to check usage outside types dir
- Types that serve as documentation (JSDoc, API contracts) = keep
- Types with zero external references AND zero re-exports = remove

**4. Legacy API routes scan:**

- Check for API routes with zero frontend references:
  - List all routes: `find src/app/api -name "route.ts"`
  - For each route, check: `grep -r "/api/routepath" src/ --include="*.ts" --include="*.tsx"` (excluding the route itself)
- Routes with zero fetch/Link/router references = flag in report (don't auto-delete -- might be webhook targets)

**5. Environment variable audit:**

- List all env vars: scan `.env`, `.env.example`, `.env.local` (if accessible)
- For each var: `grep -r "VARNAME" src/ --include="*.ts" --include="*.tsx"` plus check `next.config.ts`
- Flag unused vars in report for user review (NEVER auto-delete per locked decision)
