  <action>
    1. Install packages:
       ```bash
       pnpm add @upstash/ratelimit @upstash/redis
       ```

    2. Create `src/lib/rate-limit/config.ts`:
       - Export a `RateLimitTier` type with categories: `auth-signin`, `auth-signup`, `api-write`, `public-read`, `driver-location`, `driver-action`, `customer`, `admin`, `global`
       - Export a `RATE_LIMITS` object mapping each tier to `{ max: number, window: string }` reading from env vars with defaults:
         - `auth-signin`: `RATE_LIMIT_AUTH_SIGNIN_MAX` (default 5) / `RATE_LIMIT_AUTH_SIGNIN_WINDOW` (default "1 m")
         - `auth-signup`: `RATE_LIMIT_AUTH_SIGNUP_MAX` (default 3) / `RATE_LIMIT_AUTH_SIGNUP_WINDOW` (default "1 h")
         - `api-write`: `RATE_LIMIT_API_WRITE_MAX` (default 10) / `RATE_LIMIT_API_WRITE_WINDOW` (default "1 m")
         - `public-read`: `RATE_LIMIT_PUBLIC_READ_MAX` (default 60) / `RATE_LIMIT_PUBLIC_READ_WINDOW` (default "1 m")
         - `driver-location`: `RATE_LIMIT_DRIVER_LOCATION_MAX` (default 2) / `RATE_LIMIT_DRIVER_LOCATION_WINDOW` (default "1 m")
         - `driver-action`: `RATE_LIMIT_DRIVER_ACTION_MAX` (default 10) / `RATE_LIMIT_DRIVER_ACTION_WINDOW` (default "1 m")
         - `customer`: `RATE_LIMIT_CUSTOMER_MAX` (default 30) / `RATE_LIMIT_CUSTOMER_WINDOW` (default "1 m")
         - `admin`: `RATE_LIMIT_ADMIN_MAX` (default 120) / `RATE_LIMIT_ADMIN_WINDOW` (default "1 m")
         - `global`: `RATE_LIMIT_GLOBAL_IP_MAX` (default 120) / `RATE_LIMIT_GLOBAL_IP_WINDOW` (default "1 m")

    3. Create `src/lib/rate-limit/client.ts`:
       - Import `Redis` from `@upstash/redis` and `Ratelimit` from `@upstash/ratelimit`
       - Import `RATE_LIMITS` from config
       - Create module-level `redis` singleton: `Redis.fromEnv()` -- but ONLY if env vars are set. If `UPSTASH_REDIS_REST_URL` is not defined, export `null` for redis and all limiters.
       - Create shared ephemeral cache: `new Map()`
       - Create and export named limiter instances, each using `Ratelimit.slidingWindow()`:
         - `authSignInLimiter` -- prefix `rl:auth-signin`, timeout 3000
         - `authSignUpLimiter` -- prefix `rl:auth-signup`, timeout 3000
         - `apiWriteLimiter` -- prefix `rl:api-write`, timeout 3000
         - `publicReadLimiter` -- prefix `rl:public-read`, timeout 3000
         - `driverLocationLimiter` -- prefix `rl:driver-location`, timeout 3000
         - `driverActionLimiter` -- prefix `rl:driver-action`, timeout 3000
         - `customerLimiter` -- prefix `rl:customer`, timeout 3000
         - `adminLimiter` -- prefix `rl:admin`, timeout 3000
         - `globalLimiter` -- prefix `rl:global`, timeout 3000
       - All limiters: `analytics: false` (avoid waitUntil dependency), `ephemeralCache: cache`
       - Type each export as `Ratelimit | null` so callers handle the disabled case

    4. Create `src/lib/rate-limit/identifiers.ts`:
       - `getClientIp(request: Request): string` -- extract from `x-forwarded-for` (first entry, trimmed), fallback to `x-real-ip`, fallback to `"unknown"`. Do NOT use `request.ip` (undefined in route handlers on Vercel).
       - `getServerActionIp(): Promise<string>` -- uses `headers()` from `next/headers` to get same headers. For use in Server Actions where there is no Request object.
       - `getIdentifier(request: Request, userId?: string): string` -- returns `userId` if provided, otherwise `getClientIp(request)`.

    5. Create `src/lib/rate-limit/check.ts`:
       - Import `NextResponse` from `next/server`
       - Import `Ratelimit` type from `@upstash/ratelimit`
       - Import `logger` from `@/lib/utils/logger`
       - Export `UserRole` type: `"customer" | "driver" | "admin" | "anon"`
       - Export interface `RateLimitOptions { limiter: Ratelimit | null; identifier: string; role: UserRole; route: string; }`
       - Export interface `RateLimitResult`:
         - `{ limited: true; response: NextResponse }` OR `{ limited: false; headers: Record<string, string> }`
       - Export async function `checkRateLimit(opts: RateLimitOptions): Promise<RateLimitResult>`:
         - If `opts.limiter` is null, return `{ limited: false, headers: {} }` (fail open when Redis not configured)
         - Call `opts.limiter.limit(opts.identifier)`
         - If `!result.success`: log via `logger.warn("Rate limit exceeded", { api: opts.route, flowId: "rate-limit", role: opts.role, identifier: opts.identifier.substring(0, 8) + "..." })` and return `{ limited: true, response: NextResponse.json({ error: { code: "RATE_LIMITED", message: "Too many requests. Please wait a moment." } }, { status: 429, headers: { "Retry-After": String(retryAfterSeconds), "X-RateLimit-Limit": String(result.limit), "X-RateLimit-Remaining": "0", "X-RateLimit-Reset": String(result.reset) } }) }`
         - If `result.success`: return `{ limited: false, headers: { "X-RateLimit-Limit": ..., "X-RateLimit-Remaining": ..., "X-RateLimit-Reset": ... } }`
       - Export async function `checkServerActionRateLimit(opts: { limiter: Ratelimit | null; identifier: string; role: UserRole; route: string }): Promise<{ limited: boolean; retryAfterSeconds?: number }>`:
         - Simplified version for Server Actions (returns plain object, not NextResponse)
         - If limiter is null, return `{ limited: false }`
         - If limited, log same as above and return `{ limited: true, retryAfterSeconds }`
         - If not limited, return `{ limited: false }`

    6. Create `src/lib/rate-limit/index.ts` barrel:
       - Re-export everything from client, config, check, identifiers
  </action>
