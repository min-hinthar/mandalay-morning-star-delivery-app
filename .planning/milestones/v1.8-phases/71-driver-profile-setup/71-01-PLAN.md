  <action>
    **1. Create migration `supabase/migrations/024_driver_photos_storage.sql`:**
    - Create `driver-photos` bucket: public=true, file_size_limit=5242880 (5MB), allowed_mime_types=['image/jpeg','image/png','image/webp']
    - Follow exact pattern from `007_menu_photos_storage.sql`
    - RLS policies:
      - INSERT: authenticated driver can upload to path starting with their driver_id (join drivers table, check user_id = auth.uid(), folder = driver_id::text). Use `(select auth.uid())` initplan wrapper.
      - SELECT: public read (anyone can view profile photos — needed for customer tracking view)
      - UPDATE: authenticated driver can update own photos (same check as INSERT)
      - DELETE: authenticated driver can delete own photos OR admin can delete any
    - Add comments on policies

    **2. Create `src/lib/supabase/driver-storage.ts`:**
    - `BUCKET = 'driver-photos'`
    - `MAX_SIZE = 5 * 1024 * 1024` (5MB)
    - `ALLOWED_TYPES = ['image/jpeg', 'image/png', 'image/webp', 'image/heic', 'image/heif']`
    - `validateDriverPhoto(file: File)`: check type (including HEIC by extension fallback) and size
    - `compressDriverPhoto(file: File)`: use `browser-image-compression` with options `{ maxSizeMB: 0.2, maxWidthOrHeight: 1024, useWebWorker: true, fileType: 'image/jpeg' }` — handles HEIC→JPEG, EXIF orientation, size reduction
    - `uploadDriverPhoto(blob: Blob, driverId: string)`: upload to `{driverId}/{Date.now()}.jpg`, return `{ path, publicUrl }`
    - `deleteDriverPhoto(path: string)`: delete from storage
    - `extractDriverPhotoPath(publicUrl: string)`: extract storage path from full URL (for deletion)
    - Install `browser-image-compression` via `pnpm add browser-image-compression`
    - Install `react-easy-crop` via `pnpm add react-easy-crop`

    **3. Update `src/lib/validations/driver.ts`:**
    - Add `driverSelfUpdateSchema` — similar to `updateDriverSchema` but with these differences:
      - `fullName`: optional string, min 2, max 100
      - `phone`: optional string, min 10 (relaxed validation, no strict regex — accept any format for Burmese numbers)
      - `vehicleType`: optional, accept full enum ('car','motorcycle','bicycle','van','truck') for backward compat
      - `licensePlate`: optional string, no format validation (per CONTEXT decision)
    - Do NOT modify existing `updateDriverSchema` (used by admin)

    **4. Create `src/app/api/driver/profile/route.ts` (PATCH):**
    - Use `requireDriver()` auth guard from `@/lib/auth`
    - Rate limit with `driverActionLimiter` (existing)
    - Parse body with `driverSelfUpdateSchema`
    - Update drivers table: `vehicle_type`, `license_plate`, `phone` (if provided)
    - Update profiles table: `full_name` (if provided) using `auth.userId`
    - Follow dual-table pattern from admin `PATCH /api/admin/drivers/[id]`
    - Update driver table FIRST (more likely to fail on enum validation), then profiles
    - Call `revalidatePath('/driver')` to bust RSC cache for dashboard completeness
    - Return merged updated data: `{ fullName, phone, vehicleType, licensePlate }`

    **5. Create `src/app/api/driver/profile/photo/route.ts`:**
    - **POST:** `requireDriver()`, rate limit, accept `multipart/form-data` with `photo` field
      - Get `request.formData()`, extract file
      - Validate file server-side (type, size) — defense in depth
      - Upload to Supabase Storage: `driver-photos/{driverId}/{timestamp}.jpg`
      - If driver has existing `profile_image_url`: extract old path, delete from storage
      - Update `drivers.profile_image_url` with new public URL
      - Call `revalidatePath('/driver')`
      - Return `{ profileImageUrl }`
    - **DELETE:** `requireDriver()`, rate limit
      - If driver has `profile_image_url`: extract path, delete from storage
      - Set `drivers.profile_image_url = null`
      - Call `revalidatePath('/driver')`
      - Return `{ success: true }`

    **6. Update `src/app/api/driver/me/route.ts`:**
    - Add `license_plate` and `created_at` to the driver SELECT query
    - Add `licensePlate` and `createdAt` to the response object
    - These are needed by the profile page (license plate editing) and "member since" display
