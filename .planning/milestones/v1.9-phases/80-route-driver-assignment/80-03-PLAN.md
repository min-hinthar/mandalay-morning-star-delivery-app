  <action>
    Create `POST /api/admin/routes/[id]/stops/reassign`:

    1. Import `reassignStopSchema` from `@/lib/validations/route`.
    2. Import `updateRouteStats` from `../helpers` (one level up).
    3. Import auth/rate-limit/logger as per existing stops route pattern.
    4. Import `ProfileCheck`, `RouteParams` from the parent `types.ts`.

    **Handler logic:**
    1. Auth + admin role check (copy pattern from sibling `route.ts`).
    2. Rate limit check.
    3. Parse body with `reassignStopSchema`.
    4. Extract `routeId` from params (this is the SOURCE route).
    5. Extract `stopId` and `targetRouteId` from body.

    **Validation checks (in order):**
    6. Verify source route exists: `supabase.from('routes').select('id, status, delivery_date').eq('id', routeId).single()`.
       - If not found: 404.
       - If status !== 'planned': 400 "Can only reassign stops from planned routes".
    7. Verify target route exists: `supabase.from('routes').select('id, status, delivery_date').eq('id', targetRouteId).single()`.
       - If not found: 404 "Target route not found".
       - If status !== 'planned': 400 "Can only reassign stops to planned routes".
    8. Verify same delivery_date: source.delivery_date === target.delivery_date.
       - If different: 400 "Source and target routes must have the same delivery date".
    9. Verify source !== target: routeId !== targetRouteId.
       - If same: 400 "Cannot reassign to the same route".
    10. Verify stop exists on source route: `supabase.from('route_stops').select('id, order_id').eq('id', stopId).eq('route_id', routeId).single()`.
        - If not found: 404 "Stop not found on this route".

    **Mutation (atomic-ish):**
    11. Check if order already exists on target route: `supabase.from('route_stops').select('id').eq('route_id', targetRouteId).eq('order_id', stop.order_id).single()`.
        - If exists: 400 "Order already on target route".
    12. Get max stop_index on target route: `supabase.from('route_stops').select('stop_index').eq('route_id', targetRouteId).order('stop_index', { ascending: false }).limit(1)`.
    13. Update the stop: `supabase.from('route_stops').update({ route_id: targetRouteId, stop_index: maxIndex + 1 }).eq('id', stopId)`.
    14. Reindex source route stops: fetch remaining stops ordered by stop_index, update each with new sequential index.
    15. Update stats for both routes: `await updateRouteStats(supabase, routeId)` and `await updateRouteStats(supabase, targetRouteId)`.
    16. Return 200 with `{ stopId, sourceRouteId: routeId, targetRouteId, message: "Stop reassigned successfully" }`.

    **Error handling:**
    - Wrap in try/catch. Log errors with `logger.exception`.
    - If update in step 13 fails, the stop may be in limbo -- log as critical. (At 20-50 orders, manual fix is acceptable; true transactions deferred.)
  </action>
