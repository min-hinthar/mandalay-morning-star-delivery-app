<task type="auto">
  <name>Task 2: Upgrade webhook handler to Svix HMAC verification with audit logging</name>
  <files>src/app/api/webhooks/resend/route.ts, src/lib/email/constants.ts</files>
  <action>
1. Install svix: `pnpm add svix`

2. Rewrite `src/app/api/webhooks/resend/route.ts`:

   **Verification (replace naive secret check):**
   - Import `Webhook` from `svix`
   - Read raw body with `request.text()` (NOT `request.json()` — svix needs raw string)
   - Extract svix headers: `svix-id`, `svix-timestamp`, `svix-signature`
   - Create `new Webhook(RESEND_WEBHOOK_SECRET)` and call `wh.verify(rawBody, headers)`
   - On verification failure: log to `webhook_audit_logs` with `signature_valid: false`, source IP from `x-forwarded-for` header, payload hash via `crypto.createHash('sha256').update(rawBody).digest('hex')`, return 401
   - On success: log to `webhook_audit_logs` with `signature_valid: true`

   **Idempotency:**
   - After verification, check `svix-id` header against `webhook_audit_logs.svix_id`
   - If already exists, return 200 (already processed)

   **Event processing (keep existing logic):**
   - Parse verified payload with `JSON.parse(rawBody)`
   - Keep existing EVENT_STATUS_MAP and notification_logs update logic
   - Keep event timeline in metadata.resend_events array
   - Add status downgrade protection: don't overwrite "delivered" with "sent" — use a STATUS_PRIORITY map: `{ sent: 1, delivered: 2, opened: 3, clicked: 4, bounced: 5, complained: 5 }` and only update if new status priority >= current

   **Always return 200** after successful verification (even if processing fails) to prevent Resend retries.

3. In `src/lib/email/constants.ts`, add error guidance map:
   ```typescript
   export const ERROR_GUIDANCE: Record<string, { label: string; guidance: string }> = {
     bounced: { label: "Bounced", guidance: "Bad email address — contact customer for correct email" },
     complained: { label: "Spam Report", guidance: "Customer marked as spam — do not resend, contact directly" },
     timeout: { label: "Timeout", guidance: "Temporary issue — safe to retry" },
     rate_limit: { label: "Rate Limited", guidance: "Too many emails — wait 1 hour then retry" },
     invalid_address: { label: "Invalid Address", guidance: "Email format invalid — verify with customer" },
     unknown: { label: "Unknown Error", guidance: "Check error details — may need developer review" },
   };
   ```

Important: Use `import { createHash } from "crypto"` for payload hashing. The crypto import is Node.js built-in, no installation needed.
