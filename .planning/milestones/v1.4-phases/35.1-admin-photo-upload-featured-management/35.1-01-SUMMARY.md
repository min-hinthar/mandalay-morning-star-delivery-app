# Phase 35.1 Plan 01: Database Foundation Summary

Storage bucket and featured sections schema for admin photo upload and homepage management.

## What Was Built

### 1. Menu Photos Storage Bucket (007_menu_photos_storage.sql)

- **Bucket:** `menu-photos` with 10MB limit, JPEG/PNG only
- **RLS Policies:**
  - INSERT: Admin only (`public.is_admin()`)
  - SELECT: Public (anyone can view)
  - UPDATE/DELETE: Admin only
- **Cascade Trigger:** `delete_menu_item_photo()` cleans up storage when menu item deleted

### 2. Featured Sections Schema (008_featured_sections.sql)

- **`featured_sections` table:**
  - id, slug (unique), name, subtitle, icon, accent_color
  - sort_order, item_count, is_visible, is_predefined
  - deleted_at (soft delete), updated_by (audit)
- **`featured_section_items` junction table:**
  - Composite key (section_id, item_id), sort_order
  - ON DELETE CASCADE for both foreign keys
- **RLS Policies:** Admin write, public read for visible sections
- **Seeded 3 predefined sections:**
  - `featured-dishes` (Featured Dishes, star icon)
  - `most-popular` (Most Popular, trending-up icon)
  - `new-arrivals` (New Arrivals, sparkles icon)

### 3. TypeScript Types

- **`src/types/featured-sections.ts`:**
  - `FeaturedSection`, `FeaturedSectionItem`
  - `FeaturedSectionWithItems`, `FeaturedSectionDraft`
  - API input types for create/update
- **`src/types/database.ts`:**
  - Row/Insert/Update interfaces for both tables
  - Added to `Database.public.Tables` type
