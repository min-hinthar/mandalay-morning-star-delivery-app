## Decisions Made

1. **Public client for token validation** - RLS policy allows public SELECT on valid tokens (not expired, not accepted, not revoked), so onboarding page can validate without authentication

2. **Service role client for user creation** - Used Supabase admin API to create auth users, bypassing RLS and auto-confirming email

3. **Auto sign-in after registration** - After successful registration, user is automatically signed in via signInWithPassword and redirected to driver dashboard

4. **Cleanup on failure** - If profile or driver record creation fails, auth user is deleted to prevent orphaned accounts
