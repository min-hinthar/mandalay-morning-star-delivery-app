**Step 1 — Add safety buffer constant:**

At the top of `src/lib/utils/delivery-dates.ts`, add:

```ts
/** Safety buffer in milliseconds — reject orders this close to cutoff to prevent DB-latency edge case */
const CUTOFF_SAFETY_BUFFER_MS = 10_000; // 10 seconds
```

**Step 2 — Modify isPastCutoff:**

Change the comparison from:
```ts
return now.getTime() > cutoff.getTime();
```

To:
```ts
// BUG-07 FIX: 10-second safety buffer prevents DB-latency edge case
// Orders submitted within buffer are rejected (same message as normal cutoff)
return now.getTime() > cutoff.getTime() - CUTOFF_SAFETY_BUFFER_MS;
```

This means:
- `now > cutoff - 10s` → past cutoff (true)
- `now <= cutoff - 10s` → not past cutoff (false)

**Step 3 — DO NOT modify getTimeUntilCutoff:**

`getTimeUntilCutoff` returns raw time remaining for UI display (countdown, urgency). The safety buffer should NOT affect the customer-facing display. Customers see the countdown to the real cutoff time; the buffer only affects the submission check via `isPastCutoff`.

`getTimeUntilCutoff` already has its own `isPastCutoff` field based on `diffMs <= 0`. This is fine — it shows the gate as "past cutoff" when the actual cutoff passes. The 10-second buffer is invisible.
