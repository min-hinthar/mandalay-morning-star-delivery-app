**Admin ratings page — ambiguous PostgREST join** (found and fixed during this audit)

- **File:** `src/app/(admin)/admin/ratings/page.tsx` line 114
- **Bug:** Query used `profiles (full_name)` but `orders` has two FK paths to `profiles` (`user_id` and `contacted_by`). PostgREST would return error 300.
- **Fix applied:** Changed to `profiles!orders_user_id_fkey (full_name)`
- **Affected REQ:** CUX-12 (admin ratings view component of rating feature)
