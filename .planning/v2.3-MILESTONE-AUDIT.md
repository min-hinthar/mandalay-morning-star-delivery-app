### Broken Wiring (1)

**Phase 116 Toast action field ↔ Phase 110 useToast persistent flag** (affects CFIX-04)

The codebase has two separate toast systems:
- `useToastV8` (new) — renders via `ToastContainer` mounted in `layout.tsx`. Supports action buttons + countdown bar. Used by cart-store, Phase 116 undo flow.
- `useToast` (legacy shadcn) — has its own state with `title`/`description`/`variant`/`persistent` shape. **No renderer mounted anywhere in the app.**

`src/components/ui/checkout/usePaymentSubmit.ts:6` imports the legacy `useToast` and calls `toast({ title, variant: "destructive", persistent: true })` on Stripe timeout. This dispatches to a dead store — the toast never appears.

**Impact:** CFIX-04's core requirement ("error state with retry option") is still delivered via `CheckoutErrorBanner` (separate prop-based state), so users can still retry. The supplemental persistent notification is dead.

**Fix:** Change `usePaymentSubmit.ts` import to `@/lib/hooks/useToastV8` and adapt the call shape (`message`/`type: "error"`/`duration: 0` — no `persistent` field exists in v8).
