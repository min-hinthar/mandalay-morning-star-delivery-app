  <files>src/app/api/admin/photos/process/route.ts, src/lib/supabase/storage.ts</files>
  <action>
1. Create `src/app/api/admin/photos/process/route.ts` — POST endpoint for server-side image processing:
   - Accept FormData with `file` field and optional `menuItemId` field
   - Validate admin auth via Supabase session check (pattern from existing `/api/admin/` routes)
   - Use `sharp` library (add to dependencies: `pnpm add sharp && pnpm add -D @types/sharp`)
   - Processing pipeline:
     a. Validate input: reject if not image/jpeg or image/png, reject if raw size > 10MB
     b. Load with sharp, get metadata (width, height)
     c. Reject if width < 800 or height < 600 (return 400 with clear error: "Image must be at least 800x600 pixels")
     d. Resize + crop to 4:3 aspect ratio: `sharp(buffer).resize({ width: 800, height: 600, fit: 'cover', position: 'centre' }).webp({ quality: 80 })` — output WebP
     e. Check processed size: reject if > 2MB (return 400)
     f. Upload processed buffer to Supabase Storage bucket `menu-photos` at path `{menuItemId}/{timestamp}.webp` or `unassigned/{timestamp}.webp`
     g. Return JSON: `{ path, publicUrl, width: 800, height: 600, size, format: 'webp' }`
   - Important: Use `createServerClient` (not browser client) with service role for storage upload. Follow existing API route patterns for auth check.

2. Update `src/lib/supabase/storage.ts`:
   - Change `ALLOWED_TYPES` to include `image/webp`: `["image/jpeg", "image/png", "image/webp"]`
   - Change `MAX_SIZE` to 2MB: `2 * 1024 * 1024`
   - Add new export `uploadMenuPhotoViaServer(file: File, menuItemId?: string): Promise<UploadResult>` that:
     a. Creates FormData with file and optional menuItemId
     b. POSTs to `/api/admin/photos/process`
     c. Returns UploadResult from response
   - Keep existing `uploadMenuPhoto` and `optimizeImage` for backward compatibility but mark as `@deprecated` with JSDoc comment pointing to `uploadMenuPhotoViaServer`
   - Update `validateFile` to accept webp and enforce 2MB max

3. Update `src/lib/utils/image-optimization.ts`:
   - Change `menuCard` preset from 16:9 (400x225) to 4:3 (400x300) to match the new photo pipeline aspect ratio
