  <action>
1. Create `scripts/seed-photos.ts`:
   - Import: `@supabase/supabase-js`, `fs` (readFileSync, readdirSync, statSync), `path`, `mime-types` (or infer from extension)
   - Use same env vars as seed-menu.ts: NEXT_PUBLIC_SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY
   - Create Supabase client with service role key (same pattern as seed-menu.ts)

   - **Step 1: Read local photos**
     - Read all files from `data/menu-photos/` directory
     - For each file: extract slug from filename (strip extension: `acacia-with-shrimp-curry.jpg` → `acacia-with-shrimp-curry`)
     - Build map: `Map<string, { filePath: string, filename: string }>`
     - Log: `Found {N} photos in data/menu-photos/`

   - **Step 2: Fetch all menu items**
     - Query `menu_items` table: select id, slug, image_url
     - Build map: `Map<string, { id: string, image_url: string | null }>`
     - Log: `Found {N} menu items in database`

   - **Step 3: Match and filter**
     - For each local photo, find matching menu item by slug
     - Skip if: no matching menu item (log warning: `No menu item for photo: {filename}`)
     - Skip if: menu item already has image_url that is NOT null AND does NOT contain 'fallback' in path (admin-uploaded photo takes precedence)
     - Collect items to seed: `Array<{ itemId, slug, filePath }>`
     - Log: `{N} items need fallback photos, {M} already have photos (skipped)`

   - **Step 4: Upload photos**
     - For each item to seed (sequential to avoid rate limits):
       a. Read file as Buffer: `readFileSync(filePath)`
       b. Determine content type from extension (`.jpg` → `image/jpeg`, `.png` → `image/png`)
       c. Upload to Supabase Storage: `menu-photos/{itemId}/fallback.{ext}` with `upsert: true`
       d. Get public URL
       e. Update menu_items row: `UPDATE menu_items SET image_url = {publicUrl} WHERE id = {itemId} AND (image_url IS NULL OR image_url LIKE '%fallback%')`
       f. Log progress: `[{i}/{total}] Seeded: {slug}`
     - On upload error: log error and continue (don't abort all)

   - **Step 5: Summary**
     - Log: `Photo seeding complete: {success}/{total} uploaded, {skipped} skipped, {failed} failed`
     - If any unmatched photos exist, list them

   - Support `--dry-run` flag: when present, log what WOULD happen without uploading
   - Support `--force` flag: when present, overwrite even items that already have photos

2. Add to `package.json` scripts:
   ```json
   "seed:photos": "tsx scripts/seed-photos.ts"
   ```
   (tsx is already a dev dependency from seed-menu.ts usage)
