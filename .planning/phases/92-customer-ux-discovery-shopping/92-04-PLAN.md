   ```
   const [hasOverflow, setHasOverflow] = useState(false);
   const [isAtBottom, setIsAtBottom] = useState(false);
   const modifierContainerRef = useRef<HTMLDivElement>(null);
   ```

3. Add useEffect to detect overflow when item/modifiers change:
   ```
   useEffect(() => {
     const el = modifierContainerRef.current;
     if (!el) return;
     // Check if content overflows
     setHasOverflow(el.scrollHeight > el.clientHeight);
     setIsAtBottom(false); // Reset on item change

     const onScroll = () => {
       // 4px threshold for "at bottom" to handle sub-pixel rounding
       setIsAtBottom(el.scrollTop + el.clientHeight >= el.scrollHeight - 4);
     };
     el.addEventListener('scroll', onScroll, { passive: true });
     return () => el.removeEventListener('scroll', onScroll);
   }, [selectedItem?.modifierGroups]);
   ```
   Use the correct dependency -- the actual item's modifier groups. If the component uses `item` prop, use `item?.modifierGroups` or `item?.id` as dependency.

4. Attach `ref={modifierContainerRef}` to the scrollable container div that wraps the modifier groups.

5. Add fade gradient overlay (per Claude's discretion, using fade gradient approach -- most common in food delivery apps like DoorDash/UberEats):
   ```
   {hasOverflow && !isAtBottom && (
     <div
       className="pointer-events-none absolute bottom-0 left-0 right-0 h-12 bg-gradient-to-t from-surface-primary to-transparent"
       aria-hidden="true"
     />
   )}
   ```
   - The gradient container must be inside a `relative` positioned parent that wraps the modifier scroll area.
   - Use `pointer-events-none` so clicks pass through.
