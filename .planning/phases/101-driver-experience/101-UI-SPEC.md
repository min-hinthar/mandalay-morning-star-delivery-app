## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical spacing |
| 3xl | 64px | Page header height (--header-height) |

Exceptions:
- Touch targets: 44px minimum (--touch-target-min), 56px primary driver actions (--touch-target-driver)
- Sticky bottom bar: 80px height (--driver-action-height) + safe-area-inset-bottom
- Accept/decline CTA buttons: min-h-[72px] (matching existing SimpleHome pattern)
- Safe area insets: `env(safe-area-inset-bottom, 0px)` on sticky bottom bar position offset

Note: StatusBadge uses 12px/600 (--text-xs) -- inherited from existing component, not a new declared size for this phase.

## Interaction Contracts

### Accept/Decline Card (Dashboard)

**Layout:** Full-width card, rounded-2xl, shadow-card, border-2 border-border.

**Structure:**
1. Header: gradient bg (from-accent-teal/10 to-secondary/10), "Route Assigned" heading, assigned badge
2. Preview section: stop count, city/area summary, estimated duration (if available)
3. Actions section:
   - Accept button: full-width, min-h-[72px], bg-green, text-xl font-semibold text-text-inverse, rounded-card-sm
   - Decline link: centered below accept, text-sm text-status-error, no background, underline on hover

**Animation:**
- Card entry: framer-motion `m.div`, opacity 0->1 + y 20->0, spring.default
- Accept tap: `active:scale-[0.98]`, success toast, card swap animation (opacity crossfade, 300ms)
- After accept: card transitions to normal RouteCard with "accepted" status pill

**States:**
- Default: both buttons enabled
- Accepting: Accept button shows Loader2 spinner, Decline link disabled
- Declining: opens ConfirmDialog with reason textarea
- Accepted: card replaced by RouteCard with "Accepted" pill + "Start Route" button

### Accept/Decline Bar (Route Page)

**Layout:** Fixed bottom bar, full-width, bg-surface-primary, shadow-nav-top, z-[30] (--zindex-fixed).

**Structure:**
- Height: 80px (--driver-action-height) + safe-area bottom inset
- Position: `fixed bottom-0 left-0 right-0`, offset by `pb-[env(safe-area-inset-bottom,0px)]`
- Inner layout: `flex gap-3 px-4 items-center`, two buttons evenly split
  - Decline button: `flex-1 h-14`, variant outline, text-status-error border-status-error
  - Accept button: `flex-1 h-14`, bg-green text-text-inverse

**Scroll behavior:** Stop list scrolls above the sticky bar. Apply `pb-[calc(80px+env(safe-area-inset-bottom,0px))]` to scrollable content to prevent overlap.

### Decline Confirmation Dialog

**Trigger:** Tapping "Decline" link/button on either AcceptDeclineCard or AcceptDeclineBar.

**Content:**
- Title: "Decline Route?"
- Description: "This route will be unassigned and the admin will be notified."
- Optional textarea: placeholder "Reason (optional)", max-length 500, 3 rows
- Confirm button: "Decline Route" (destructive variant)
- Cancel button: "Keep Route" (outline variant)
- Loading state: confirm button shows spinner during API call

**Animation:** Modal fade-in (existing Modal component handles this).

### Driver Stop Reorder (Advanced Mode)

**Visibility:** Always visible on reorderable stops (pending + enroute). No toggle.

**Desktop (>= md breakpoint):**
- DragHandle (grip icon) on left side of each reorderable stop, `aria-label="Drag to reorder"`
- Drag interaction: @dnd-kit sortable, visual lift + shadow during drag
- Delivered/skipped stops: no handle, visually muted (opacity-50)

**Mobile (< md breakpoint):**
- MoveButtons (chevron-up / chevron-down) on right side of each reorderable stop
- Up button: `aria-label="Move stop up"`, down button: `aria-label="Move stop down"`
- Each button: min 44px touch target
- First item: up button disabled. Last item: down button disabled.

**Save behavior:** Immediate save per drag-drop. Optimistic UI update. Error toast + revert on failure.

## Animation Contract

All animations use existing framer-motion `m` component + `useAnimationPreference()` guard.

| Element | Animation | Parameters |
|---------|-----------|------------|
| Accept card entry | Fade + slide up | `opacity: 0->1, y: 20->0`, spring.default |
| Accept card exit (on accept) | Fade out + scale | `opacity: 1->0, scale: 1->0.95`, duration 200ms |
| Route card entry (after accept) | Fade + slide up | `opacity: 0->1, y: 20->0`, spring.default, delay 150ms |
| Sticky bottom bar entry | Slide up from bottom | `y: 100%->0`, spring.default |
| Drag item lift | Scale + shadow | @dnd-kit default overlay, scale 1.02, shadow-lg |
| Status badge pulse | Soft scale oscillation | Existing ACTIVE_STATUSES animation (5 cycles) |

All animations disabled when `shouldAnimate === false` (reduced motion preference).
