**2. Wire auto-price-update in useCartValidation (D-20):**

In `useCartValidation.ts`, after the validation loop completes (where `priceChangedIds` are populated), add an effect that auto-calls `updateItemPrice()` for each price-changed item.

Add a new `useEffect` after the existing effects, BEFORE the `useMemo` return:
```typescript
// Phase 115 D-20: Auto-update cart prices when validation detects changes.
// This closes the gap between "show PriceChangeBadge" and "fix the cart" —
// cart always reflects server truth after validation, reducing checkout friction.
const updateItemPrice = useCartStore((s) => s.updateItemPrice);

useEffect(() => {
  if (!menuData?.data?.categories || items.length === 0) return;

  const categories = menuData.data.categories;
  const menuLookup = buildMenuLookup(categories);

  for (const cartItem of items) {
    const found = menuLookup.get(cartItem.menuItemId);
    if (found && found.item.basePriceCents !== cartItem.basePriceCents) {
      updateItemPrice(cartItem.cartItemId, found.item.basePriceCents);
    }
  }
}, [menuData, items, updateItemPrice]);
```

This runs whenever `menuData` updates (on mount refetch or 3-min poll). For each cart item with a price mismatch, it calls `updateItemPrice()` to sync. The PriceChangeBadge will still show briefly (validation result includes price-changed status), but the cart total now reflects the correct price.

IMPORTANT: The `updateItemPrice` selector must use `useCartStore((s) => s.updateItemPrice)` — direct selector, not `getState()` in useMemo (gotcha H1).
