**Request params (D-04):**

**Auth (D-05):**

**Query construction:**
```typescript
import { NextRequest, NextResponse } from "next/server";
import { z } from "zod";
import { createClient } from "@/lib/supabase/server";
import { logger } from "@/lib/utils/logger";
import { checkRateLimit, authenticatedLimiter, getClientIp } from "@/lib/rate-limit";
import type { OrderStatus } from "@/types/order";
import type { RefundStatus } from "@/types/database";

const paramsSchema = z.object({
  cursor: z.string().optional(),
  limit: z.coerce.number().min(1).max(50).default(10),
});

interface OrderRow {
  id: string;
  status: OrderStatus;
  refund_status: RefundStatus;
  total_cents: number;
  delivery_window_start: string | null;
  placed_at: string;
  order_items: Array<{ quantity: number }>;
}

interface DecodedCursor {
  placed_at: string;
  id: string;
}
```

**Cursor decode (D-02):**
```typescript
function decodeCursor(cursor: string): DecodedCursor | null {
  try {
    const decoded = JSON.parse(atob(cursor));
    if (typeof decoded.placed_at === "string" && typeof decoded.id === "string") {
      return decoded as DecodedCursor;
    }
    return null;
  } catch {
    return null;
  }
}

function encodeCursor(row: { placed_at: string; id: string }): string {
  return btoa(JSON.stringify({ placed_at: row.placed_at, id: row.id }));
}
```

**GET handler:**
1. Rate limit with `authenticatedLimiter`
2. Auth via `supabase.auth.getUser()` — return 401 if no user
3. Parse + validate params via `paramsSchema`
4. Build query: `.from("orders").select("id, status, refund_status, total_cents, delivery_window_start, placed_at, order_items (quantity)").eq("user_id", user.id).order("placed_at", { ascending: false }).order("id", { ascending: false })`
5. If cursor provided and valid: add `.lt("placed_at", decodedCursor.placed_at)` OR use compound filter: `.or(\`placed_at.lt.${cursor.placed_at},and(placed_at.eq.${cursor.placed_at},id.lt.${cursor.id})\`)`
6. Fetch `limit + 1` rows via `.limit(limit + 1)` (D-03: detect hasMore without count query)
7. If fetched rows > limit: `hasMore = true`, slice to `limit`; else `hasMore = false`
8. Build `nextCursor` from last returned row: `encodeCursor({ placed_at: lastRow.placed_at, id: lastRow.id })`
9. Map rows to camelCase (same transform as current page.tsx): `{ id, status, refundStatus, totalCents, deliveryWindowStart, placedAt, itemCount }`

**Response shape (D-06):**
```typescript
return NextResponse.json({
  data: orders,
  pagination: {
    nextCursor: hasMore ? encodeCursor(lastRow) : null,
    hasMore,
    limit,
  },
});
```

**Error handling:** Wrap in try/catch, log with `logger.exception`, return 500 with `{ error: { code: "INTERNAL_ERROR", message: "Failed to fetch orders" } }`.

Do NOT use `profiles!orders_user_id_fkey` join — this is a customer-facing route, no profile data needed (unlike admin route). RLS handles the user_id filter.
