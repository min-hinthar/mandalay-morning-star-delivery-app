  <action>
Verify that font configuration uses `display: "swap"` which is required for IMAGE-07.

Check the Inter and Playfair_Display font configurations in layout.tsx:

```typescript
const inter = Inter({
  variable: "--font-inter",
  subsets: ["latin"],
  display: "swap",  // MUST be present
  preload: true,
});

const playfair = Playfair_Display({
  variable: "--font-playfair",
  subsets: ["latin"],
  display: "swap",  // MUST be present
  preload: true,
  weight: ["400", "600", "700"],
});
```

Run verification:
```bash
grep -n "display.*swap" src/app/layout.tsx
```

Expected output should show both fonts have `display: "swap"`.

Why: display: swap ensures text is visible immediately with system font, then swaps to custom font when loaded. This prevents FOIT (Flash of Invisible Text) and improves perceived performance.

If not present (unlikely based on research), add `display: "swap"` to font configurations.
  </action>

Grep output shows `display: "swap"` for both Inter and Playfair_Display fonts.


Run verification:
```bash
grep -n "useJsApiLoader" src/components/ui/coverage/CoverageRouteMap.tsx
```

Expected: useJsApiLoader is used, which loads Google Maps API asynchronously when the component mounts.

Additionally verify the component is in HowItWorksSection (below the fold):
```bash
grep -n "CoverageRouteMap" src/components/ui/homepage/HowItWorksSection.tsx
```

Why: Google Maps API is ~369KB. Loading it asynchronously via useJsApiLoader means it doesn't block initial page render. The component is in HowItWorksSection which is below the hero, so it loads after critical content.

Note: useJsApiLoader loads on component mount, which happens when HowItWorksSection enters the viewport. This is effectively deferred loading.
