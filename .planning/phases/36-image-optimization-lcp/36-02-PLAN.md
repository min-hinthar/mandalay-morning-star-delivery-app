  <action>
Update CardImage to include:
1. Add responsive sizes prop for browser srcset optimization
2. Add shimmer skeleton placeholder while image loads
3. Add fade-in transition when image loads
4. Add error handling to show emoji fallback on load error

Add state for tracking load status:

```typescript
const [isLoaded, setIsLoaded] = useState(false);
const [hasError, setHasError] = useState(false);
```

Update the img element:

```typescript
{imageUrl && !hasError ? (
  <img
    src={imageUrl}
    alt={alt}
    className={cn(
      "w-full h-full object-cover",
      "transition-opacity duration-150",
      isLoaded ? "opacity-100" : "opacity-0"
    )}
    loading={priority ? "eager" : "lazy"}
    sizes="(max-width: 640px) 50vw, (max-width: 1024px) 33vw, 25vw"
    onLoad={() => setIsLoaded(true)}
    onError={() => setHasError(true)}
  />
) : (
  // Emoji fallback (existing implementation)
  <div className="w-full h-full flex items-center justify-center bg-gradient-surface">
    <span className="text-5xl" role="img" aria-label={alt}>
      {getCategoryEmoji(categorySlug)}
    </span>
  </div>
)}
```

Add shimmer placeholder (shows while loading and on error shows emoji):

```typescript
{/* Shimmer placeholder - shows until image loads */}
{imageUrl && !isLoaded && !hasError && (
  <div className="absolute inset-0 bg-surface-tertiary animate-pulse" />
)}
```

Note: Keep using plain `<img>` tag as CardImage already does. The research notes it's "reliable across all devices and URL types" including Google Drive. The sizes attribute still helps browsers choose the right resource from any srcset the server provides.

Why: Shimmer prevents jarring appearance. Sizes attribute helps browser pre-calculate layout. Error fallback ensures graceful degradation.
</action>
