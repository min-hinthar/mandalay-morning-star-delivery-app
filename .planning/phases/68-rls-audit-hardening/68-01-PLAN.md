  <action>
Create `supabase/migrations/022_rls_audit_hardening.sql` with the following sections. All changes use `DROP POLICY IF EXISTS` then `CREATE POLICY` for idempotency.

**Section 1: app_settings -- Add public/anon read**
Per user decision: anon users can read app_settings (public storefront).
- `DROP POLICY IF EXISTS app_settings_admin_select ON app_settings;`
- Create `app_settings_select` policy: `FOR SELECT USING (true)` -- no `TO` clause means all roles including anon can read. This replaces the admin-only SELECT.
- Admin write policies (insert, update) remain unchanged -- do NOT touch them.

**Section 2: order_audit_log -- Admin-only access**
Per user decision: admin-only, customers never see raw audit data.
- Drop customer SELECT: `DROP POLICY IF EXISTS "Customers can view own order audit logs" ON order_audit_log;`
- Drop old admin SELECT: `DROP POLICY IF EXISTS "Admins can view all audit logs" ON order_audit_log;`
- Create `order_audit_log_select`: `FOR SELECT TO authenticated USING ((select public.is_admin()))` -- uses initplan optimization, replaces raw `EXISTS (SELECT 1 FROM profiles ...)` anti-pattern.
- Drop open INSERT: `DROP POLICY IF EXISTS "Authenticated users can insert audit logs" ON order_audit_log;`
- Create `order_audit_log_insert`: `FOR INSERT TO authenticated WITH CHECK ((select public.is_admin()))` -- restricts to admin only. Service-role bypasses RLS so server-side audit inserts via `createServiceClient()` still work.

**Section 3: customer_settings -- Add admin UPDATE**
Per user decision: admin has full read/write to all customer settings for support.
- Drop existing update policy: `DROP POLICY IF EXISTS customer_settings_update ON customer_settings;`
- Create `customer_settings_update`: `FOR UPDATE TO authenticated USING (user_id = (select auth.uid()) OR (select public.is_admin())) WITH CHECK (user_id = (select auth.uid()) OR (select public.is_admin()))` -- adds admin write capability while preserving customer own-update.

**Section 4: driver_badges -- Add UPDATE policy**
Research found missing UPDATE policy. Badge metadata (icon, name) may need admin correction.
- Create `driver_badges_update_admin`: `FOR UPDATE TO authenticated USING ((select public.is_admin())) WITH CHECK ((select public.is_admin()))` -- admin-only UPDATE.
- Also add initplan wrapper to existing policies by dropping and recreating:
  - Drop `driver_badges_select_own`, recreate with `(select public.get_my_driver_id())`
  - Drop `driver_badges_select_admin`, recreate with `(select public.is_admin())`
  - Drop `driver_badges_insert_admin`, recreate with `(select public.is_admin())`
  - Drop `driver_badges_delete_admin`, recreate with `(select public.is_admin())`

**Section 5: featured_sections -- Verify anon access**
Current policy has no `TO` clause which defaults to PUBLIC (includes anon). This is correct per PostgreSQL behavior. However, add an explicit comment confirming intent.
- Drop and recreate `featured_sections_select` with same logic but add `-- Anon + authenticated: visible sections; admin: all including soft-deleted` comment.
- Ensure the policy still uses: `USING ((is_visible = true AND deleted_at IS NULL) OR (select public.is_admin()))` -- add initplan wrapper to `is_admin()` if not already present.
- Same treatment for `featured_section_items_select`: drop and recreate with initplan wrapper on `is_admin()`.

**Section 6: Initplan optimization pass**
For policies in migrations 008-021 that use bare `public.is_admin()` or `public.get_my_driver_id()` without `(select ...)` wrapper, drop and recreate with the wrapper. Specifically check:
- `featured_sections` write policies (insert, update, delete) -- wrap `is_admin()` in `(select ...)`
- `featured_section_items` write policies -- same
- `customer_settings_select` -- already uses `(SELECT auth.uid())` but check `is_admin()` wrapping
- `customer_settings_insert` -- already uses `(SELECT auth.uid())`

For each, the pattern is:
```sql
DROP POLICY IF EXISTS "policy_name" ON table_name;
CREATE POLICY "policy_name" ON table_name
  FOR operation TO authenticated
  USING/WITH CHECK ((select public.is_admin()));
```
