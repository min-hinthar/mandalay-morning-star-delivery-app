    Keep `"use client"` directive. Keep `next/dynamic` with `ssr: false` pattern.

    For each lazy chart, the `loading` prop must use `LoadingWithTimeout` wrapping `ChartSkeleton` to implement the 10-second timeout:

    ```tsx
    loading: () => (
      <LoadingWithTimeout
        skeleton={<ChartSkeleton label="Loading revenue chart..." height={300} />}
        timeoutMs={10000}
        timeoutMessage="Charts taking longer than expected"
      />
    ),
    ```

    Chart-specific labels and heights:

    | Chart | Label | Height |
    |-------|-------|--------|
    | LazyDeliverySuccessChart | "Loading delivery success chart..." | 300 |
    | LazyETAAccuracyGauge | "Loading ETA accuracy..." | 200 |
    | LazyPeakHoursChart | "Loading peak hours chart..." | 250 |
    | LazyExceptionBreakdown | "Loading exception breakdown..." | 250 |
    | LazyPerformanceChart | "Loading performance chart..." | 300 |
    | **LazyRevenueChart** (NEW) | "Loading revenue chart..." | 300 |

    All charts use `timeoutMs={10000}` and `timeoutMessage="Charts taking longer than expected"`.

    For each dynamic import, wrap the import function with `importWithRetry`:
    ```
    () => importWithRetry(
      () => import("./DeliverySuccessChart").then(mod => mod.DeliverySuccessChart),
      "DeliverySuccessChart"
    )
    ```

    **Add LazyRevenueChart:**
    ```tsx
    export const LazyRevenueChart = dynamic(
      () => importWithRetry(
        () => import("../RevenueChart").then(mod => mod.RevenueChart),
        "RevenueChart"
      ),
      {
        loading: () => (
          <LoadingWithTimeout
            skeleton={<ChartSkeleton label="Loading revenue chart..." height={300} />}
            timeoutMs={10000}
            timeoutMessage="Charts taking longer than expected"
          />
        ),
        ssr: false,
      }
    );
    ```

    Note: RevenueChart is one directory UP from analytics (`../RevenueChart`).

    **IMPORTANT pitfall:** The admin/page.tsx is a SERVER component. `next/dynamic` with `ssr: false` cannot be used directly in server components. LazyRevenueChart (being in a `"use client"` file) is fine -- the server component just imports and renders it. The `"use client"` boundary is at LazyCharts.tsx, not at the page.

    **Update analytics/index.ts:** Add `LazyRevenueChart` to the LazyCharts re-exports:
    ```
    export {
      LazyDeliverySuccessChart,
      LazyETAAccuracyGauge,
      LazyPeakHoursChart,
      LazyExceptionBreakdown,
      LazyPerformanceChart,
      LazyRevenueChart,
    } from "./LazyCharts";
    ```
  </action>
