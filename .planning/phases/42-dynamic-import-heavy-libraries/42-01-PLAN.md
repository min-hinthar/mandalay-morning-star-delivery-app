  <action>
    **useViewportTrigger.ts:**
    Create a reusable hook that returns `{ ref, triggered }`. When the element referenced by `ref` enters the viewport (threshold: 0), set `triggered` to `true` and disconnect the observer (load persistence -- never unload).

    Options interface:
    - `fallbackToEager?: boolean` (default `true`) -- if `IntersectionObserver` is undefined, immediately set `triggered = true`
    - `threshold?: number` (default `0`)
    - `eager?: boolean` (default `false`) -- bypass viewport detection entirely, set `triggered = true` immediately (for tracking page, checkout page)

    Must be `"use client"`. Use `useRef<HTMLDivElement>`, `useState<boolean>`, `useEffect`. Pattern matches the existing IntersectionObserver usage in `CoverageRouteMap.tsx`.

    **useDynamicImportWithRetry.ts:**
    Create an async utility function `importWithRetry` (NOT a hook -- it's a pure async function used inside `next/dynamic` factory):

    ```
    async function importWithRetry<T>(
      importFn: () => Promise<T>,
      componentName: string,
      maxRetries?: number  // default 3
    ): Promise<T>
    ```

    Retry delays: `[1000, 2000, 4000]` (exponential backoff per CONTEXT.md).
    On each failure: call `logger.exception(error, { flowId: "dynamic-import", api: componentName })`.
    After final failure: throw the error (let the calling component's error boundary handle it).

    Import `logger` from `@/lib/utils/logger`.

    Do NOT make this a React hook. It's a utility function that wraps dynamic import calls.
  </action>
  <verify>
    `pnpm typecheck` passes. Both files export correctly. `useViewportTrigger` has proper `"use client"` directive. `importWithRetry` is a named export.
  </verify>
  <done>
    `useViewportTrigger` hook exists with eager/viewport/fallback modes. `importWithRetry` utility exists with exponential backoff, Sentry logging via logger.exception, and max 3 retries.
  </done>

  <action>
    **LoadingWithTimeout.tsx:**
    `"use client"` component in `src/components/ui/LoadingWithTimeout.tsx`. This is the shared timeout wrapper used by both chart and map dynamic imports.

    Props interface:
    ```
    interface LoadingWithTimeoutProps {
      skeleton: React.ReactNode;       // The skeleton to show initially (ChartSkeleton or MapSkeleton)
      timeoutMs: number;               // Threshold in ms (10000 for charts, 15000 for maps)
      timeoutMessage?: string;         // e.g. "Charts taking longer than expected"
      className?: string;
    }
    ```

    Implementation:
    - `const [timedOut, setTimedOut] = useState(false);`
    - `useEffect` with `setTimeout(() => setTimedOut(true), timeoutMs)` and cleanup via `clearTimeout` on unmount
    - Before timeout: render `{skeleton}` only
    - After timeout: render the skeleton PLUS a timeout overlay/message below it:
      ```jsx
      <div className={cn("relative", className)}>
        {skeleton}
        {timedOut && (
          <div className="mt-2 flex items-center justify-center gap-2 text-sm text-text-muted">
            <Clock className="h-4 w-4" />
            <span>{timeoutMessage ?? "Taking longer than expected..."}</span>
            <Button
              variant="ghost"
              size="sm"
              onClick={() => window.location.reload()}
              className="ml-1"
            >
              <RefreshCw className="h-3 w-3 mr-1" />
              Retry
            </Button>
          </div>
        )}
      </div>
      ```

    Import `Clock`, `RefreshCw` from `lucide-react`, `Button` from `@/components/ui/button`, `cn` from `@/lib/utils/cn`.

    The retry button triggers `window.location.reload()` which remounts the component tree and re-triggers the dynamic import. This is intentional -- if the import is stuck (not failing, just slow), a page reload is the most reliable recovery.

    **ChartSkeleton.tsx:**
    `"use client"` component with props: `{ label: string; height?: number; className?: string }`.

    Renders faux bar chart shapes (12 bars of varying heights: `[40, 65, 50, 80, 55, 70, 45, 75, 60, 85, 50, 70]`) using `flex items-end gap-2`. Each bar is a `div` with `flex-1 rounded-t bg-surface-tertiary animate-pulse` and `animationDelay: ${i * 80}ms` for staggered pulse.

    Outer container: `rounded-xl bg-surface-primary p-6 shadow-sm` with `role="status"` and `aria-label={label}`.
    Bar area height: `height - 48` pixels (subtract padding).
    Below bars: `<p className="mt-3 text-xs text-text-muted text-center">{label}</p>` for the per-chart label (e.g., "Loading revenue chart...").

    Use `cn()` from `@/lib/utils/cn` for className merging.

    **ChartErrorCard.tsx:**
    `"use client"` component with props: `{ message: string; onRetry?: () => void; isFinal?: boolean; height?: number; className?: string }`.

    Match `RouteError.tsx` styling pattern:

    - Center content vertically in container with `minHeight: height ?? 200`.

    - If `isFinal`: show `<p className="text-xs text-text-muted">Unable to load. Please refresh the page.</p>`.

    Import `AlertTriangle`, `RefreshCw` from `lucide-react`, `Button` from `@/components/ui/button`, `motion` from `framer-motion`, `cn` from `@/lib/utils/cn`.
