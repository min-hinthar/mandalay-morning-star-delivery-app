<task type="auto">
  <name>Task 3: Create driver onboard API endpoint</name>
  <files>src/app/api/driver/onboard/route.ts</files>
  <action>
    Create POST endpoint at `/api/driver/onboard`:

    **Request body:**
    ```typescript
    {
      token: string;
      fullName: string;
      phone: string;
      vehicleType: 'car' | 'motorcycle' | 'bicycle' | 'van' | 'truck';
      licensePlate: string;
      password: string;
    }
    ```

    **Processing steps:**
    1. Validate token in driver_invites (pending, not expired)
    2. Get email from invite record
    3. Create Supabase auth user with email + password
    4. Create profiles record with role='driver'
    5. Create drivers record with vehicle info
    6. Update driver_invites: set accepted_at = NOW()
    7. Sign in user with signInWithPassword
    8. Return 201 with redirect URL to driver dashboard

    **Error handling:**
    - 400: Invalid input data
    - 404: Token not found or expired
    - 409: Email already registered
    - 500: Registration failed (rollback if possible)

    **Security:**
    - Token check happens first before any user creation
    - Use transaction or sequential checks with cleanup on failure

  </action>
  <verify>Endpoint compiles, handles all error cases</verify>
  <done>
    - Valid token + form data creates driver account
    - Driver is automatically signed in
    - Invite marked as accepted
  </done>
