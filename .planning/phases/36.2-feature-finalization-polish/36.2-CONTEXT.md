<decisions>
## Implementation Decisions

### Driver Invites

**Invite method:**

- Email only — admin enters email, system sends invite link
- Resend invites from the Pending tab
- Revoke pending invites supported

**Onboarding flow:**

- Full registration: driver fills name, phone, vehicle info, then sets password
- Token expires after 24 hours
- After registration: auto-login directly to driver dashboard

**UI location:**

- "Invite Driver" button on /admin/drivers page
- Separate "Pending" tab alongside "Active" and "Archived" tabs
- Pending tab shows pending invites with resend/revoke actions

**Email provider:**

- Resend for sending invite emails

### Route Optimization

**Trigger:**

- Auto-optimize when route is created
- Manual "Optimize" button for re-optimization

**Algorithm:**

- Google Routes API with traffic data
- Fallback behavior: Claude's discretion on error handling

**Display:**

- Before/after comparison view
- Show time and distance savings ("Saves 15 min / 3.2 miles")
- Admin clicks "Apply" to confirm optimized order

**Manual override:**

- Allow drag-and-drop reordering after optimization
- Show "Not optimized" warning badge when manually reordered

**Time windows:**

- Per-stop delivery windows considered in optimization
- Fixed time slots: Morning (9-12), Afternoon (12-5), Evening (5-8)
- Time slots configurable by admin in /admin/settings
- Customers set preferred window in account profile, can override at checkout

### Orders Manage/Track Tab

**Available actions:**

- Full CRUD: status change, driver assign, edit items, cancel, refund

**View Full Details:**

- Expand in-place (row expands to show details inline)
- Comprehensive info: items, total, customer name/phone/address, payment method, driver info, notes, status history, audit log

**Status changes:**

- Status action buttons: "Mark Preparing", "Mark Ready", etc.

**Driver assignment:**

- Smart suggestions based on driver location and availability

**Item editing:**

- Full edit capability: add, remove, change quantities
- Total auto-recalculates when items change
- Customer notified of changes

**Refunds:**

- Both options available: per-item refund OR custom amount entry

**Audit trail:**

- Reason required for all edits (not just refunds/cancels)

**Communication:**

- Call/text buttons to contact customer directly from order view

### Settings & Account Polish

**Validation:**

- Inline errors below each invalid field + summary toast on submit
- Validation runs on submit only (not on blur/change)

**Loading states:**

- Button spinner while saving (button disabled)
- Skeleton loaders for all data loading states

**Success feedback:**

- Checkmark on save button + toast message

**Unsaved changes:**

- Custom styled modal: "Discard changes?" with Save/Discard options

**Server errors:**

- Map known errors to relevant fields inline
- Generic toast for unknown errors

**Save behavior:**

- Explicit save button required (no auto-save)

**Scope:**

- All admin settings tabs polished equally (Delivery, Operations, Notifications)
- All customer account tabs polished equally (Profile, Orders, Addresses)

**Empty states:**

- Illustrations + helpful text message
- Illustration style: Claude's discretion based on existing design

### Claude's Discretion

- Illustration style for empty states
- Fallback behavior when Google Routes API fails
- Exact skeleton loader designs per component
- Email template design for driver invites

</decisions>
