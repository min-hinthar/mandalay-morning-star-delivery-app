# Phase 36.2 Plan 01: Email Infrastructure for Driver Invites Summary

Resend email service with React Email template and driver_invites database table for token-based driver onboarding.

## What Was Built

### 1. Email Dependencies (Task 1)

Installed:

- `resend@6.9.1` - Email API client for transactional emails
- `@react-email/components@1.0.6` - Type-safe React components for templates

### 2. Email Service Wrapper (Task 2)

**File:** `src/lib/services/email.ts`

Exports:

- `sendDriverInvite(email: string, token: string, expiresAt: Date)`

Features:

- Resend client initialization with `RESEND_API_KEY`
- Configurable from address via `EMAIL_FROM_ADDRESS` env var
- URL generation with `NEXT_PUBLIC_BASE_URL`
- Returns `{ data, error }` from Resend API

### 3. Driver Invite Email Template (Task 2)

**File:** `src/emails/DriverInviteEmail.tsx`

Props:

- `inviteUrl: string` - Full onboarding URL with token
- `expiresAt: string` - Formatted expiration date

Content:

- Morning Star branding
- Welcome message for driver team
- "Accept Invitation" button (primary red #C8102E)
- Expiration notice in footer

### 4. Database Migration (Task 3)

**File:** `supabase/migrations/012_driver_invites.sql`

Table: `driver_invites`
| Column | Type | Purpose |
|--------|------|---------|
| id | UUID | Primary key |
| email | TEXT | Invitee email address |
| token | TEXT UNIQUE | Onboarding URL token |
| invited_by | UUID | Admin who sent invite |
| expires_at | TIMESTAMPTZ | Token expiration |
| accepted_at | TIMESTAMPTZ | When driver registered |
| revoked_at | TIMESTAMPTZ | If admin revoked |
| created_at | TIMESTAMPTZ | Invite creation time |

Indexes:

- `idx_driver_invites_token` - Fast token lookup during onboarding
- `idx_driver_invites_email` - Check for existing invites

RLS Policies:

- Admins can manage all invites (SELECT, INSERT, UPDATE, DELETE)
- Public can SELECT valid tokens (not expired, not accepted, not revoked)
