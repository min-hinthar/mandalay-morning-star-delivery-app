  <files>
    src/components/ui/account/SettingsTab/useCustomerSettings.ts

  <action>
    Create `src/components/ui/account/SettingsTab/useCustomerSettings.ts` with `'use client'` directive.

    **Interface:** Hook returns:
    ```typescript
    {
      settings: CustomerSettings | null;
      setSettings: React.Dispatch<React.SetStateAction<CustomerSettings | null>>;
      isLoading: boolean;
      isSaving: boolean;
      hasChanges: boolean;
      error: string | null;
      save: () => Promise<void>;
      discard: () => void;
      updateField: <K extends keyof CustomerSettings>(key: K, value: CustomerSettings[K]) => void;
    }
    ```

    **Implementation:**
    1. `settings` state and `originalSettings` state (both `CustomerSettings | null`)
    2. `useEffect` on mount: `fetch("/api/account/settings")` → on success, set both `settings` and `originalSettings` from `response.data`
    3. `hasChanges` computed via `useMemo`: deep compare `settings` vs `originalSettings` using `JSON.stringify`
    4. `updateField` helper: `setSettings(prev => prev ? { ...prev, [key]: value } : prev)`
    5. `save` function:
       - Set `isSaving = true`
       - Build partial payload with only changed fields (compare each key of `settings` vs `originalSettings`)
       - Convert to snake_case for API: `dietaryRestrictions` → `dietary_restrictions`, etc.
       - Also include `customAllergies` merged into `dietary_restrictions` array: combine `settings.dietaryRestrictions` + `settings.customAllergies` into a single `dietary_restrictions` array for the API (the DB stores all restrictions in one JSONB array)
       - `PATCH /api/account/settings` with JSON body
       - On success: update `originalSettings` to match current `settings`
       - Set `isSaving = false`
    6. `discard` function: reset `settings` to `originalSettings`
    7. Error handling: set `error` state on fetch/save failures, clear on success

    **Key detail:** The API stores dietary restrictions + custom allergies in one `dietary_restrictions` JSONB array. The hook splits them on load: predefined options go to `dietaryRestrictions`, unrecognized values go to `customAllergies`. On save, they merge back.

    Import `CustomerSettings`, `DEFAULT_CUSTOMER_SETTINGS`, `DIETARY_OPTIONS` from `./settings-types`.

    - Hook exports `useCustomerSettings` function
  </verify>
