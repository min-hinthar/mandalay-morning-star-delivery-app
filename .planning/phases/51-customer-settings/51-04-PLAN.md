  <action>
    Both files need `'use client'` directive.

    **useFontSize.ts:**

    1. Export `FONT_SIZES` const: `{ small: 14, medium: 16, large: 18 } as const`
    2. Export `FontSize` type: `keyof typeof FONT_SIZES`
    3. Export `useFontSize()` hook:
       - State: `size` (FontSize), default `"medium"`
       - `useEffect` on mount: read `localStorage.getItem("font-size")`, validate it's a valid key, set state. Also apply CSS variable immediately: `document.documentElement.style.setProperty("--font-size-base", FONT_SIZES[stored] + "px")`
       - `setFontSize(newSize: FontSize)` function:
         - `localStorage.setItem("font-size", newSize)`
         - `setSize(newSize)`
         - `document.documentElement.style.setProperty("--font-size-base", FONT_SIZES[newSize] + "px")`
       - Return `{ size, setFontSize, sizes: FONT_SIZES }`
    4. Hydration-safe: use `useEffect` for reads, not `useState(() => localStorage...)`. Render with default until hydrated.

    **useSoundPreference.ts:**

    Simple wrapper that reads/writes the existing `"soundEffectsEnabled"` localStorage key (same key as `useSoundEffect.ts`).

    1. Export `useSoundPreference()` hook:
       - State: `isEnabled` boolean, default `true`
       - `useEffect` on mount: read `localStorage.getItem("soundEffectsEnabled")`, set to `value !== "false"`
       - `setEnabled(enabled: boolean)` function:
         - `localStorage.setItem("soundEffectsEnabled", String(enabled))`
         - `setEnabled(enabled)`
       - Return `{ isEnabled, setEnabled }`
    2. This hook is separate from `useSoundEffect` because it's a simpler toggle without AudioContext. The Display section only needs the preference state, not the play function.
    3. Uses the SAME localStorage key as `useSoundEffect` (`"soundEffectsEnabled"`) so they stay in sync.
