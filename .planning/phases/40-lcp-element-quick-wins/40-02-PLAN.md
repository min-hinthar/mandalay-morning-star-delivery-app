<task type="auto">
  <name>Task 1: Convert CardImage to Next.js Image</name>
  <files>src/components/ui/menu/UnifiedMenuItemCard/CardImage.tsx</files>
  <action>
Replace the raw `<img>` tag with Next.js Image component:

1. Add import: `import Image from "next/image";`

2. Update the props interface to use priority meaningfully (not just for API compatibility)

3. Replace the img tag inside motion.div with:
```tsx
<Image
  src={imageUrl}
  alt={alt}
  fill
  className="object-cover"
  loading={priority ? "eager" : "lazy"}
  fetchPriority={priority ? "high" : undefined}
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 50vw, 33vw"
  onError={() => setHasError(true)}
/>
```

4. IMPORTANT: The Image component must be inside the motion.div for parallax to work. The `fill` prop works with position:absolute which motion.div already has (`className="absolute inset-0"`).

5. Remove the eslint-disable comment for no-img-element since we're now using Next.js Image.

6. Verify remotePatterns in next.config.ts covers:
   - `**.supabase.co` (Supabase storage)
   - `drive.google.com` (Google Drive)
   - `lh3.googleusercontent.com` (Google image proxy)

7. The Image component needs `unoptimized` prop if URL is from external source that doesn't support Next.js image optimization. Check if needed based on remotePatterns config.

Note: Next.js 16 deprecated `priority` prop in favor of `loading="eager"` + `fetchPriority="high"`. Do NOT use `priority={true}`.

Handle potential issues:
- If image fails to load due to URL format, fall back to emoji (existing onError handler)
- Motion wrapper should continue to work since Image uses fill positioning
