GAP-100-01: In useReorderStops.ts, the `onError` callback receives `reorderedStops` (the NEW stops) instead of the PREVIOUS stops. The caller `RouteDetailClient.tsx:145-147` expects to receive the pre-mutation stops to revert `localStops`.

Fix: capture the previous stops before the mutation. Change `handleReorder` to accept a second parameter or capture stops before mutation. The cleanest fix is to store previous stops before the fetch:
```ts
const handleReorder = useCallback(
  async (reorderedStops: StopDetail[]) => {
    setIsReordering(true);

    const stopOrder = reorderedStops.map((stop, index) => ({
      stopId: stop.id,
      stopIndex: index,
    }));

    const body: Record<string, unknown> = { stopOrder };
    if (routeStatus === "in_progress") {
      body.forceOverride = true;
    }

    try {
      const response = await fetch(`/api/admin/routes/${routeId}`, {
        method: "PATCH",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(body),
      });

      if (!response.ok) {
        onError(reorderedStops);
        toast({ message: "Failed to reorder stops", type: "error" });
        return;
      }

      onSuccess();
    } catch {
      onError(reorderedStops);
      toast({ message: "Failed to reorder stops", type: "error" });
    } finally {
      setIsReordering(false);
    }
  },
  [routeId, routeStatus, onSuccess, onError]
);
```

```ts
const reorderStops = useReorderStops({
```

The caller sets `localStops` to the new order BEFORE calling `handleReorder`. Then on error, the callback `(previousStops) => setLocalStops(previousStops)` should receive the OLD stops to revert. But `useReorderStops.handleReorder` passes `reorderedStops` (the new order) to `onError`.

The fix: The hook needs to accept the previous stops as a parameter, or the caller needs to capture them. The simplest fix is to update the hook interface to accept `previousStops` as a second parameter to `handleReorder`:

Change the hook signature and implementation:
```ts
const handleReorder = useCallback(
  async (reorderedStops: StopDetail[], previousStops: StopDetail[]) => {
    setIsReordering(true);
    // ... same fetch logic ...
    // On error:
    onError(previousStops);  // was: onError(reorderedStops)
  },
  [routeId, routeStatus, onSuccess, onError]
);
```

Then update the caller in RouteDetailClient.tsx `handleReorder`:
```ts
const handleReorder = useCallback(
  (reorderedStops: StopDetail[]) => {
    const previousStops = [...localStops];
    const updated = reorderedStops.map((stop, i) => ({ ...stop, stopIndex: i }));
    setLocalStops(updated);
    reorderStops.handleReorder(updated, previousStops);
  },
  [reorderStops, localStops]

```
