1. Fetch orders with profiles, order_items, route_stops
2. Fetch notification_logs separately with `.in("order_id", orderIds)`

**New pattern (1 query):**
Include `notification_logs` in the main query select string. Supabase PostgREST supports this since notification_logs has a foreign key `order_id` referencing `orders(id)`.

```typescript
const { data: orders, error: ordersError } = await supabase
  .from("orders")
  .select(
    `
    id,
    status,
    refund_status,
    total_cents,
    delivery_window_start,
    placed_at,
    needs_contact,
    order_items (quantity),
    profiles (
      full_name,
      email
    ),
    route_stops (id),
    notification_logs (status, created_at)
  `
  )
  .order("placed_at", { ascending: false })
  .limit(200)
  .returns<OrderRow[]>();
```

Update the `OrderRow` interface to include:
```typescript
notification_logs: Array<{ status: string; created_at: string }> | null;
```

Remove the `EmailLogRow` interface and the second query block entirely (the `if (orderIds.length > 0)` block that fetches notification_logs separately).

For EACH of the 5 files, apply these changes to the GET handler:

**1. Accept `request: Request` parameter** (some handlers currently take no args):
- `admin/orders/route.ts`: Change `export async function GET()` to `export async function GET(request: Request)`
- `admin/drivers/route.ts`: Change `export async function GET()` to `export async function GET(request: Request)`
- `admin/menu/route.ts`: Change `export async function GET()` to `export async function GET(request: Request)`
- `admin/categories/route.ts`: Change `export async function GET()` to `export async function GET(request: Request)`
- `admin/routes/route.ts`: Already has `request: NextRequest` -- good.

**2. Parse pagination query params:**
```typescript
const { searchParams } = new URL(request.url);
const page = Math.max(1, parseInt(searchParams.get("page") || "1", 10));
const limit = Math.min(100, Math.max(1, parseInt(searchParams.get("limit") || "25", 10)));
```
Note: Default limit is 25 per CONTEXT.md decision.

**3. Add `{ count: "exact" }` to select:**
Change `.select("...")` to `.select("...", { count: "exact" })`.

**4. Replace `.limit(N)` with `.range()`:**
```typescript
const rangeStart = (page - 1) * limit;
const rangeEnd = rangeStart + limit - 1;
// ... query chain
.range(rangeStart, rangeEnd)
```

**5. Extract count from response:**
```typescript
const { data, error, count } = await query;
```

**6. Return paginated response shape:**
```typescript
return NextResponse.json({
  data: mappedData, // or data ?? []
  pagination: {
    page,
    limit,
    total: count ?? 0,
    totalPages: Math.ceil((count ?? 0) / limit),
  },
});
```
