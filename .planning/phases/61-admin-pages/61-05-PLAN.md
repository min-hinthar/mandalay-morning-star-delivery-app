**2. TiptapEditor** (`TiptapEditor.tsx`):

`'use client'` wrapper around Tiptap's `useEditor` hook:
- Extensions: `StarterKit` (bold, italic, bullet list, ordered list, heading, blockquote) + `Placeholder` (configurable placeholder text) + `Link` (autolink)
- Toolbar: row of icon buttons for bold, italic, bullet list, ordered list, link

- Toolbar button active state: highlighted when formatting is active
- Editor content area with min-height ~150px
- Styled with design tokens: border, rounded, focus ring
- Props: `content` (initial HTML string), `onChange` (returns HTML string), `placeholder`
- Export `getHTML()` accessor for parent to read content

**3. ManualEmailDialog** (`ManualEmailDialog.tsx`):

Two-step modal dialog for composing and sending manual emails:

Props: `open`, `onClose`, `orderId`, `orderNumber` (for display), `customerEmail`, `orderSummary` (items + delivery details for auto-footer)

**Step 1 — Compose:**
- Subject line text input (required)
- TiptapEditor for body content
- Auto-generated footer preview showing: "Regarding Order #XXXX: [items list], Delivery: [address]"
- "Preview Email" button to advance to step 2

**Step 2 — Confirmation Preview:**
- Shows subject, formatted body (rendered HTML), auto-footer
- Recipient: customer email displayed
- "Back to Edit" and "Send Email" buttons
- "Send Email" button with loading spinner

On send:
1. POST to `/api/admin/emails/compose` with `{ orderId, subject, htmlBody, recipientEmail }`
2. On success: toast success, close dialog, trigger EmailHistory refresh (via callback prop or parent refetch)
3. On error: toast error, stay on preview step

  <action>
Create `POST /api/admin/emails/compose` route:

1. `requireAdmin()` auth check
2. Validate body with Zod:
```typescript
const composeSchema = z.object({
  orderId: z.string().uuid(),
  subject: z.string().min(1).max(200),
  htmlBody: z.string().min(1).max(50000),
  recipientEmail: z.string().email(),
});
```

3. Fetch order basic info from `orders` table to build the auto-footer:
   - Order ID, items (names + quantities), delivery address
   - Format into a simple footer HTML block

4. Compose the full email:
   - Wrap `htmlBody` + auto-footer in the existing email layout using `buildEmailElement` from `@/lib/email/build`
   - If `buildEmailElement` expects specific template types, create a generic wrapper or use `dangerouslySetInnerHTML` within the email layout
   - Check how existing email templates work in `src/lib/email/build.ts` — follow the same pattern

5. Send via `sendEmail` from `@/lib/email/send`:
   - `to`: recipientEmail
   - `subject`: from request body
   - `html`: composed email HTML
   - `notificationType`: "manual" (or closest match to existing types)
   - `orderId`: from request body

6. Log to `notification_logs` table (the sendEmail function may handle this automatically — check)

7. Return `{ success: true, emailId: result.id }` on success, `{ error: "..." }` on failure

**Sanitization:** The htmlBody comes from Tiptap which produces safe HTML (no script tags). Still, consider basic sanitization or trust Tiptap's output. Do NOT use a full sanitizer library — Tiptap's output is inherently safe.
