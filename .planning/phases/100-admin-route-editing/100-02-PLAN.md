    split_route(p_source_route_id uuid, p_stop_ids uuid[], p_new_driver_id uuid DEFAULT NULL) RETURNS uuid:
    - Validate source route exists (SELECT INTO STRICT)
    - Validate all stop IDs belong to source route
    - Validate at least 1 stop remains in source after split
    - Create new route with status 'planned', inherit delivery_date
    - UPDATE route_stops SET route_id = new_route for selected stops, assign sequential stop_index
    - Reindex source route remaining stops
    - Update stats for both routes
    - Return new route ID

    merge_routes(p_destination_route_id uuid, p_source_route_id uuid) RETURNS int:
    - Validate source route is 'planned' status
    - Validate destination route exists
    - Get max stop_index in destination
    - UPDATE source stops to destination route_id, append after last stop
    - DELETE source route
    - Update destination stats
    - Return total stop count
