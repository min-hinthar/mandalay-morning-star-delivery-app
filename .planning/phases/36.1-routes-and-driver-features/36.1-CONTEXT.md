<decisions>
## Implementation Decisions

### Driver Detail Page
- **Layout focus**: Activity-focused — recent routes and deliveries as main view, profile info secondary
- **Actions available**: Full management — edit profile, toggle status, assign to route, view/clear exceptions, permanently archive
- **History depth**: Last 7 days of activity by default, load more on demand
- **Location display**: Show last known location, not live tracking (live tracking is on route page)
- **Deactivation**: Archive with reason required (soft delete, keeps history)
- **Ratings display**: Average rating plus list of recent customer feedback/reviews
- **Driver-specific stats**: Rating avg, deliveries count, recent routes visible

### Route Detail Page
- **Layout**: Map-first — large map showing route polyline and stops, stop list below
- **Stop status changes**: Admin override allowed with reason required (audit trail)
- **Stop editing**: Full edit — reorder, add orders to route, remove orders from route
- **Exception handling**: Full management — view, resolve, reschedule delivery, contact customer
- **Live tracking**: Periodic refresh (every 30 seconds or on-demand) when route in progress
- **Stop card info**: Detailed — address, status, customer name/phone, order items summary, delivery time window, notes, proof photos, exception details
- **Driver reassignment**: Can reassign anytime, even mid-route
- **Status control**: Admin can force all transitions (planned, in_progress, completed, cancelled)
- **Route stats**: Shown at top of page — total stops, completed, pending, total distance, estimated time
- **Driver contact**: Click-to-call/SMS buttons directly on route page
- **Map provider**: Google Maps
- **Proof photos**: View inline — thumbnails in stop card, click to enlarge
- **Route optimization**: Available with preview before applying reordered stops

### Settings Page
- **Scope**: Delivery + Operations settings (not full business config)
- **Layout**: Tabbed interface — tabs for Delivery, Drivers/Operations, Notifications
- **Save behavior**: Explicit save button with unsaved changes warning
- **Storage**: All settings in Supabase database table (changeable without redeploy)
- **Validation**: Strict — validate all fields, prevent saving if invalid
- **Restore defaults**: Yes — single global button to restore all to defaults

**Delivery settings (Standard):**
- Delivery radius
- Minimum order amount
- Free delivery threshold
- Time windows
- Delivery fee tiers
- Cut-off times

**Driver/Operations settings (Standard):**
- Default vehicle types
- Max deliveries per route
- Auto-assignment rules
- Route optimization preferences

**Notification settings:**
- Channels + toggles — configure which channels (email, SMS, push) per notification type

### Driver-Route Workflow
- **Assignment method**: Auto-assign option with manual override — can auto-assign based on availability or manually select
- **Auto-assign logic**: Availability only (first available active driver)
- **Driver scheduling**: Simple on/off — drivers either active or inactive, no complex scheduling
- **Driver notifications**: Full — assignment + status changes + new stops added
- **Concurrent routes**: One at a time — driver must complete current route before starting another
- **Decline/no response**: Manual reassign — admin notified, must manually reassign
- **Route creation**: Flexible — can assign during creation or leave unassigned
- **Dispatch visibility**: Enhanced routes list with driver assignment visibility (not separate page)

### Customer Account Page
- **Access**: Header dropdown menu links to /account page
- **Layout**: Tabbed — tabs for Profile, Orders, Addresses, Payment
- **Scope**: Full management — profile (name/email/phone), order history, saved addresses, payment methods
- **Reorder**: One-click — "Reorder" button adds same items to cart immediately
- **Address limit**: 5 saved addresses max
- **Order history**: All time visible (complete history since account creation)
- **Account deletion**: Request only — submit request, admin processes deletion

### Admin Account Page
- **Scope**: Profile only — name, email, profile photo (no security settings here)

### Order Actions & Cancellation
- **Cancel access**: Both with rules — customer can cancel until preparation starts, admin can cancel anytime
- **Cancel window**: Until order status changes to 'preparing'
- **Cancel reason**: Required for both customer and admin
- **Admin actions**: Full edit — change status, edit items, change address, adjust totals, refund
- **Edit notifications**: Admin choice — admin can choose whether to notify customer on each edit
- **Refunds**: Item-level — select specific items to refund
- **Audit trail**: Major actions only — log status changes, cancellations, refunds
- **Actions location**: Both — quick actions in order list row, full actions on detail page
- **Bulk actions**: Yes — full bulk ops (status changes, print receipts/labels, assign to route, cancel)

### Claude's Discretion
- Driver profile photo editing (view-only vs admin upload)
- Admin internal notes system for drivers
- Settings audit log
- Exact notification channel configurations
