## Decisions Made

- **JSONB values:** Used JSONB for value column to support numbers, strings, booleans, and arrays without type columns
- **No delete RLS:** Settings should never be deleted individually - restore uses upsert pattern
- **Separate restore route:** Created `/restore` endpoint rather than DELETE method for clarity
- **Record<string, unknown> response:** Used flexible type for API response to simplify dynamic key assignment
