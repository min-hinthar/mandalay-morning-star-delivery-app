## Decisions Made
- **Reorder returns cart items, not creates order:** Server validates availability and warns about changes, client handles cart state
- **Cancel window is status-based:** Only pending/confirmed orders cancellable by customer, not time-based
- **Cancellation reason in special_instructions:** Appends to existing notes with timestamp marker
- **Address limit at API level:** Returns 400 with ADDRESS_LIMIT code when exceeding 5
