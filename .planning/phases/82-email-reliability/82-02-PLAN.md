```typescript
// Step 7: Flag order for manual customer contact
try {
  await supabase
    .from("orders")
    .update({ needs_contact: true })
    .eq("id", options.orderId);

  logger.warn("Order flagged for manual contact after email failure", {
    flowId,
    orderId: options.orderId,
    userId: options.userId,
  });
} catch (flagErr) {
  // Non-blocking — email failure is already logged
  logger.error("Failed to flag order for manual contact", {
    flowId,
    orderId: options.orderId,
  });
  logger.exception(flagErr, { flowId, orderId: options.orderId });
}
```

Important: The needs_contact update uses `.eq("id", options.orderId)` — this is idempotent. Multiple failed emails for the same order just set `true` again. No race condition risk.

Create `POST /api/admin/orders/[id]/contact` endpoint:

1. Use standard admin auth pattern: `requireAdmin()` + `checkRateLimit()`

3. Verify the order exists and has `needs_contact = true`
4. Update the order:
   ```typescript

     .from("orders")
     .update({
       needs_contact: false,
       contacted_at: new Date().toISOString(),
       contacted_by: auth.userId,
     })
     .eq("id", orderId)
     .eq("needs_contact", true); // Only update if still flagged
   ```
5. Log the resolution action to the order's audit_logs:
   ```typescript
   await supabase.from("audit_logs").insert({
     order_id: orderId,
     actor_id: auth.userId,
     actor_role: "admin",
     action: "marked_contacted",
     reason: "Manual customer contact after email delivery failure",
   });
   ```
6. Return `{ success: true, contactedAt: new Date().toISOString() }`

Error responses:
- 404 if order not found
- 400 if order is not flagged (needs_contact is false)
- Standard 500 for unexpected errors

Follow the exact pattern from `src/app/api/admin/emails/[id]/resend/route.ts` for structure and error handling.
