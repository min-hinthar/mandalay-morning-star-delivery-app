### Reduced Motion

When `data-reduce-motion="true"` on `<html>`:
- All shimmer animations disabled (animation: none)
- Stagger delays zeroed (animation-delay: 0s)
- Skeleton renders as static `bg-surface-tertiary` fill
- Crossfade transitions still play (opacity only, not motion)

## Loading Hierarchy (LOAD-05)

| Tier | Component | Duration | Trigger | Fallback |
|------|-----------|----------|---------|----------|
| 1. Skeleton | Content-shaped skeleton with shimmer | 0-15s (10 cycles) | Page load (loading.tsx / Suspense) | Tier 2 |
| 2. Spinner + message | BrandedSpinner + "Taking longer than expected..." | 15-30s | LoadingWithTimeout timeoutMs=15000 | Tier 3 |
| 3. Timeout error | Clock icon + message + Retry button | 30s+ | Second timeout or manual | Page reload |

Enforcement: Wrap all page-level skeletons with `LoadingWithTimeout` at `timeoutMs={15000}`.

## Offline State Contract

### IDB-First Menu Flow (LOAD-04)

| State | What User Sees | Visual Treatment |
|-------|----------------|------------------|
| Cold start, IDB cache exists | Menu with cached data | StaleBadge above menu grid: "Cached {relative time}" in amber |
| Cold start, IDB cache exists, network succeeds | Fresh menu replaces cached | StaleBadge fades out over 0.3s; content crossfades via SkeletonCrossfade |
| Cold start, IDB cache exists, network fails | Cached menu persists | StaleBadge remains; OfflineBanner shows at top |
| Cold start, no IDB cache, network fails | Offline fallback page | Existing /offline precached page |
| IDB cache > 24h stale | Treat as no cache | Network-first only; stale IDB ignored |

StaleBadge placement: Single badge above menu grid, NOT per-item. Uses `Badge variant="status-warning"` with Clock icon.

### Cart Sync on Reconnect (CFIX-08)

| Scenario | User Notification | Visual Component |
|----------|-------------------|------------------|
| All prices match | Toast: "Cart synced successfully" | Standard toast, auto-dismiss 3s |
| Price changed on N items | Banner: "N items updated since you were offline" + item list | Reuse CheckoutErrorBanner pattern from Phase 111 |
| Item unavailable | Toast: "{Item name} is no longer available and was removed" | Persistent toast (useToast({ persistent: true })) |
| Mixed changes | Banner for price changes + toast per removed item | Banner + toasts stack |
