The current code has a subtle race condition: `shouldDebounce()` reads and writes the external `recentAdditions` Map BEFORE `set()` is called. Two rapid calls can both pass `shouldDebounce()` before either reaches `set()`.

**Fix: Move debounce tracking inside the Zustand `set()` callback.**

The `recentAdditions` Map is module-level and shared, but the check happens outside `set()`. The fix is to move the debounce check inside the `set()` atomic updater, using a pattern where `set()` returns unchanged state if debounced.

However, Zustand `set()` with `(state) =>` is synchronous and the debounce Map is module-scoped, so the actual race condition is between:
1. Call A: `shouldDebounce` returns false, sets timestamp
2. Call B (< 300ms later): `shouldDebounce` returns true

This actually works correctly for the synchronous case because JavaScript is single-threaded. The real risk is if `addItem` is called from async contexts where the event loop interleaves between `shouldDebounce()` and `set()`.

**The safer approach:** Move the debounce check AND timestamp update inside `set()`:

```ts
addItem: (item) => {
  const signature = createItemSignature(item);

  // BUG-06 FIX: Debounce check inside set() for atomicity
  set((state) => {
    const now = Date.now();
    const lastAdd = recentAdditions.get(signature);

    if (lastAdd && now - lastAdd < DEBOUNCE_MS) {
      console.debug("[cart] Debounced duplicate add:", signature);
      return state; // Return unchanged state
    }

    // Update timestamp atomically with state change
    recentAdditions.set(signature, now);

    // Clean old entries periodically
    if (recentAdditions.size > 100) {
      const cutoff = now - DEBOUNCE_MS * 2;
      for (const [key, time] of recentAdditions.entries()) {
        if (time < cutoff) {
          recentAdditions.delete(key);
        }
      }
    }

    const { items } = state;

    // ... rest of existing addItem logic (find existing, merge or add new)
    // Keep ALL existing logic from the current set() callback
  });
},
```

Remove the standalone `shouldDebounce()` call that currently precedes `set()`. The `shouldDebounce` function can remain exported for testing, but the `addItem` method no longer calls it — the check is inline inside `set()`.
