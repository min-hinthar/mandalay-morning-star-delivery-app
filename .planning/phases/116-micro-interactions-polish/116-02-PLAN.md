2. Add swipe hint logic inside the component, gated on `isFirstItem && shouldAnimate`:

```typescript
// Swipe hint state
const [showSwipeHint, setShowSwipeHint] = useState(false);

useEffect(() => {
  if (!isFirstItem || !shouldAnimate) return;

  // One-time hint (per D-15)
  try {
    if (localStorage.getItem("swipeHintSeen")) return;
  } catch {
    return; // SSR or storage blocked
  }

  // Delay 800ms after mount (per D-14)
  const timer = setTimeout(() => {
    setShowSwipeHint(true);
  }, 800);

  return () => clearTimeout(timer);
}, [isFirstItem, shouldAnimate]);

// Mark hint as seen after animation completes
const handleSwipeHintComplete = useCallback(() => {
  setShowSwipeHint(false);
  try {
    localStorage.setItem("swipeHintSeen", "1");
  } catch {
    // Storage blocked
  }
}, []);
```

3. Apply the bounce animation to the draggable `m.div`. When `showSwipeHint` is true, use `animate` prop with keyframes:

```tsx
<m.div
  drag={shouldAnimate && !isStale ? "x" : false}
  // ... existing drag props ...
  animate={showSwipeHint ? { x: [0, -30, 0] } : undefined}
  transition={showSwipeHint ? spring.ultraBouncy : springConfig}
  onAnimationComplete={() => {
    if (showSwipeHint) handleSwipeHintComplete();
  }}
  // ... rest unchanged ...
>
```

Important: The `animate` prop for the hint must NOT conflict with the drag behavior. The hint fires once on mount, then the drag takes over. After `handleSwipeHintComplete`, `showSwipeHint` is false and `animate` is `undefined`, so drag works normally.

```typescript
useEffect(() => {

  updateFadeIndicators();

  const resizeObserver = new ResizeObserver(updateFadeIndicators);
```
