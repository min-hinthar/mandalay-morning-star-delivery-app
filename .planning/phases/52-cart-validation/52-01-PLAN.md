    export interface CartItemValidation {

      newPriceCents?: number;       // present when status is 'price-changed'

<task type="auto">
  <name>Task 2: Create useCartHydrated and useCartValidation hooks</name>
  <files>src/lib/hooks/useCartValidation.ts</files>
  <action>
    Create `src/lib/hooks/useCartValidation.ts` with two exported hooks:

    **useCartHydrated():**
    - Returns boolean indicating if Zustand persist has rehydrated
    - Uses `useCartStore.persist.hasHydrated()` for initial check
    - Subscribes to `useCartStore.persist.onFinishHydration()` for async case
    - Cleans up subscription on unmount
    - Pattern from research (Zustand v5 official API)

    **useCartValidation():**
    - Returns `CartValidationResult`
    - Gates on `useCartHydrated()` -- returns `{ status: 'idle', ... }` until hydrated
    - Uses `useMenu()` for menu data. Force-refetch on mount by calling `refetch()` after initial data load (ensures freshness for cart validation, not cached stale data)
    - When both hydrated AND menu data available, runs validation:
      1. Build a `Map<menuItemId, { item: MenuItem, categoryId: string }>` from all categories
      2. For each cart item:
         - If menuItemId NOT found in menu map AND item is not in any category: mark `unavailable`
         - If found but `isSoldOut === true`: mark `sold-out`
         - If found but `basePriceCents !== cartItem.basePriceCents`: mark `price-changed` with direction
         - Otherwise: mark `valid`
      3. For sold-out and unavailable items, compute suggestions:
         - Use `cartItem.categoryId` if present to find the category
         - Fallback: find the category containing this menuItemId in the menu data
         - Return up to 3 active, non-sold-out items from the same category (excluding current item)
      4. Compute `hasBlockingIssues = soldOutIds.length > 0 || unavailableIds.length > 0`
    - If menu fetch errors: return `{ status: 'error', ... }` with empty arrays (silent fail per decision -- allow checkout)
    - Memoize validation result with `useMemo` keyed on `items` array + `menuData`
    - Status transitions: idle -> validating -> done (or error)

    **Error resilience:** If useMenu() fails, status = 'error', all arrays empty, hasBlockingIssues = false (silent fail, backend validates on submit per locked decision).
  </action>
  <verify>Run `pnpm typecheck` -- no type errors. Hook exports are importable. Manual inspection: validation logic handles all 4 states (valid, sold-out, unavailable, price-changed).</verify>
  <done>useCartHydrated returns hydration state. useCartValidation compares cart items against live menu data after hydration, returning categorized results with suggestions. Silent fail on API error.</done>
</task>
