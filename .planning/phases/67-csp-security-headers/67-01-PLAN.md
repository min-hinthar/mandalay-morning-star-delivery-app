<task type="auto">
  <name>Task 1: Add CSP Report-Only and security headers to next.config.ts</name>
  <files>next.config.ts</files>
  <action>
In the existing `headers()` function in `next.config.ts`, add a new entry at the TOP of the returned array that matches `/(.*)`  (all routes). This entry must include:

**CSP directives (single global policy):**
- `default-src 'self'`
- `script-src 'self' 'unsafe-inline' https://maps.googleapis.com` -- add `'unsafe-eval'` ONLY when `process.env.NODE_ENV === "development"` (React Fast Refresh). Never ship `unsafe-eval` in production.
- `style-src 'self' 'unsafe-inline' https://fonts.googleapis.com` -- `'unsafe-inline'` is permanent (Framer Motion injects `<style>` elements)
- `img-src 'self' blob: data: https://*.supabase.co https://lh3.googleusercontent.com https://drive.google.com https://maps.googleapis.com https://maps.gstatic.com`
- `font-src 'self' https://fonts.gstatic.com`
- `connect-src 'self' https://*.supabase.co wss://*.supabase.co https://*.ingest.us.sentry.io https://maps.googleapis.com https://routes.googleapis.com`
- `worker-src 'self'`
- `object-src 'none'`
- `base-uri 'self'`
- `form-action 'self'`
- `frame-ancestors 'none'`
- `upgrade-insecure-requests` -- ONLY in production (gated by `!isDev`). Breaks localhost HTTP if included in dev.
- `report-uri {SENTRY_CSP_ENDPOINT}` -- construct endpoint from Sentry DSN (see below)
- `report-to csp-endpoint`

**Sentry CSP endpoint construction:**
Parse `NEXT_PUBLIC_SENTRY_DSN` env var at build time. The DSN format is `https://<PUBLIC_KEY>@<ORG_SLUG>.ingest.us.sentry.io/<PROJECT_ID>`. Extract public key, org ingest domain, and project ID. Build the endpoint URL: `https://<ORG_SLUG>.ingest.us.sentry.io/api/<PROJECT_ID>/security/?sentry_key=<PUBLIC_KEY>`.

If `NEXT_PUBLIC_SENTRY_DSN` is not set, omit `report-uri` and `report-to` directives (graceful fallback for local dev without Sentry).

**Header key:** `Content-Security-Policy-Report-Only` (NOT enforcing yet -- Phase success criterion #4 upgrades to enforcing after validation period).

**Additional headers on same `/(.*)`  entry:**
- `Report-To`: JSON object with `group: "csp-endpoint"`, `max_age: 10886400`, `endpoints` array pointing to same Sentry URL, `include_subdomains: true`
- `Reporting-Endpoints`: `csp-endpoint="{SENTRY_URL}"` (modern browsers)
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(), geolocation=(self)`
- `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload`

**Keep existing headers** for `/api/health`, `/fonts/:path*`, `/icons/:path*` -- they remain as separate entries AFTER the global entry.

**Code organization:** Define `isDev`, CSP directive array, and Sentry endpoint parsing ABOVE the `headers()` function or at the top of it. Use `.filter(Boolean).join("; ")` to build the CSP string.

Do NOT create a `middleware.ts` file. Do NOT use nonces (pointless with `unsafe-inline`).
  </action>

Run `pnpm build` -- must succeed without errors. The CSP header string should be visible in build output or inspectable via `curl -I localhost:3000` in dev mode.

Run `pnpm lint` to verify no lint issues introduced in next.config.ts.
  </verify>
  <done>
next.config.ts has a `/(.*)`  header entry with Content-Security-Policy-Report-Only containing all 8 external domains, Report-To/Reporting-Endpoints pointing to Sentry, and 5 security headers. Existing health/fonts/icons headers preserved. Build passes.
  </done>
