  <name>Task 1: Create useUpdateBanner hook with interaction-aware countdown</name>
  <files>src/lib/hooks/useUpdateBanner.ts</files>
  <action>
  Create `src/lib/hooks/useUpdateBanner.ts` with the following logic:

  **State:**
  - `showBanner: boolean` - whether to display the banner
  - `countdown: number` - current countdown value (starts at 10)
  - `isPaused: boolean` - countdown paused due to user interaction
  - `dismissCount: number` - track dismissals (persisted in sessionStorage)
  - `version: string` - from `process.env.NEXT_PUBLIC_APP_VERSION || ""`
  - `waitingWorker: ServiceWorker | null` - reference to waiting SW

  **Core behavior:**
  1. Detect waiting service worker (same as current UpdatePrompt detection logic)
  2. 10-second countdown timer (1s interval), decrements when not paused
  3. When countdown hits 0: post `SKIP_WAITING` message to waiting worker
  4. Listen for `controllerchange` -> `window.location.reload()`

  **Interaction pause (locked decision):**
  - Listen for `scroll`, `keydown`, `touchstart`, `mousedown` events (passive)
  - On any interaction: set `isPaused = true`
  - After 3 seconds of no interaction: resume countdown (`isPaused = false`)
  - Use a debounced idle timer that resets on each interaction

  **Dismissal tracking (locked decision):**
  - `handleDismiss()`: increment `dismissCount`, hide banner, save count to `sessionStorage`
  - Banner re-shows on next navigation via `usePathname` change detection
  - After 3 dismissals: `canDismiss = false` (force-reload, no dismiss button)

  **Page deferral (locked decision):**
  - If `pathname` is `/cart` or `/checkout` (or starts with): suppress banner entirely
  - Store `hasDeferredUpdate = true`
  - When pathname changes to non-deferred page: show banner

  **Vibration (locked decision):**
  - On first show: `navigator.vibrate?.(100)` (subtle 100ms vibration)

  **Post-update toast:**
  - Before reload, set `sessionStorage.setItem("mms-sw-updated", "true")`
  - In the hook (or separate effect): on mount, check sessionStorage for flag, if found:
    - Show toast: "Updated to latest version!" (use existing toast system)
    - Remove the flag

  **Return type:**
  ```ts
  interface UseUpdateBannerReturn {
    showBanner: boolean;
    countdown: number;
    isPaused: boolean;
    canDismiss: boolean;
    version: string;
    progress: number; // 0-1 fraction for progress bar (countdown / TOTAL)
    handleDismiss: () => void;
    handleUpdateNow: () => void;
  }
  ```

  Add `'use client'` directive. Export the hook and add to `src/lib/hooks/index.ts` barrel export.
  </action>

  `pnpm typecheck` passes. Hook file exists with correct exports. No lint errors.
  </verify>
  <done>
  useUpdateBanner hook handles 10s countdown with interaction pause, dismissal tracking (force after 3), page deferral (/cart, /checkout), mobile vibration, and post-update toast flag.
  </done>
