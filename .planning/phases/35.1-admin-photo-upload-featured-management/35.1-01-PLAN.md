<task type="auto">
  <name>Task 2: Create featured sections schema migration</name>
  <files>supabase/migrations/008_featured_sections.sql</files>
  <action>
Create SQL migration for featured sections:

1. `featured_sections` table:
   - id: uuid primary key (gen_random_uuid())
   - slug: text unique not null
   - name: text not null
   - subtitle: text nullable (optional description)
   - icon: text nullable (lucide icon name)
   - accent_color: text nullable (hex color)
   - sort_order: integer not null default 0
   - item_count: integer not null default 6 (how many items to show)
   - is_visible: boolean not null default true
   - is_predefined: boolean not null default false (for Featured, Popular, New)
   - deleted_at: timestamptz nullable (soft delete)
   - created_at: timestamptz default now()
   - updated_at: timestamptz default now()
   - updated_by: uuid references profiles(id) nullable

2. `featured_section_items` junction table:
   - section_id: uuid references featured_sections(id) on delete cascade
   - item_id: uuid references menu_items(id) on delete cascade
   - sort_order: integer not null default 0
   - primary key (section_id, item_id)

3. RLS policies (follow patterns from 002_rls_policies.sql):
   - SELECT: Public for visible sections, admin sees all (including soft-deleted)
   - INSERT/UPDATE/DELETE: Admin only

4. Seed predefined sections:
   - 'featured-dishes' (name: "Featured Dishes", is_predefined: true, sort_order: 0)
   - 'most-popular' (name: "Most Popular", is_predefined: true, sort_order: 1)
   - 'new-arrivals' (name: "New Arrivals", is_predefined: true, sort_order: 2)

5. Add updated_at trigger using existing pattern.
  </action>
  <verify>SQL syntax valid. Tables have proper constraints and indexes.</verify>
  <done>Featured sections schema ready. Three predefined sections seeded.</done>
</task>
