  <files>src/app/api/admin/sections/route.ts, src/app/api/admin/sections/[id]/route.ts, src/app/api/admin/sections/[id]/items/route.ts, src/app/api/admin/sections/reorder/route.ts, src/app/api/admin/sections/most-popular/suggest/route.ts</files>
  <action>
1. `src/app/api/admin/sections/route.ts`:

GET: List all sections (including soft-deleted for admin)

- Include item count per section
- Order by sort_order
- Return: FeaturedSection[] with itemCount

POST: Create new section

- Body: { name, subtitle?, icon?, accentColor?, itemCount? }
- Auto-generate slug from name
- Set sort_order to max + 1
- Return created section

2. `src/app/api/admin/sections/[id]/route.ts`:

GET: Get single section with items

- Include menu_items via junction table
- Return: FeaturedSectionWithItems

PATCH: Update section

- Body: Partial<FeaturedSection>
- Cannot change is_predefined
- Update updated_at, updated_by
- Return updated section

DELETE: Soft delete section

- Set deleted_at = now()
- Predefined sections: set is_visible = false instead of soft delete
- Return: { success: true }

POST (with action=restore): Restore soft-deleted section

- Clear deleted_at
- Return restored section

POST (with action=duplicate): Duplicate section

- Copy section with new slug (append -copy)
- Copy all item assignments
- Return new section

3. `src/app/api/admin/sections/[id]/items/route.ts`:

GET: List items in section

- Order by sort_order
- Include full MenuItem data
- Return: MenuItem[]

POST: Add items to section

- Body: { itemIds: string[] }
- Assign sort_order incrementally
- Return updated item list

DELETE: Remove item from section

- Query param: ?itemId=xxx
- Delete from junction table
- Return: { success: true }

PATCH: Reorder items within section

- Body: { itemIds: string[] } (in new order)
- Update sort_order for each
- Return: { success: true }

4. `src/app/api/admin/sections/reorder/route.ts`:

POST: Reorder sections

- Body: { sectionIds: string[] } (in new order)
- Update sort_order for each
- Return: { success: true }

5. `src/app/api/admin/sections/most-popular/suggest/route.ts`:

GET: Auto-suggest popular items based on order statistics

- Query orders table to count item occurrences (sum of order_items.quantity per item_id)
- Group by item_id, order by total quantity DESC
- Limit to top 20 items
- Return: { items: MenuItem[], suggestedAt: timestamp }
- This enables "Most Popular" section to show auto-suggested items that admin can review and override

All routes require admin auth. Follow existing patterns from menu API.
