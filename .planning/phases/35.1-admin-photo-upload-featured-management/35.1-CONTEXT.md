# Phase 35.1: Admin Photo Upload & Featured Management - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Enable admin photo uploads to Supabase Storage, remove Browse All Dishes section from homepage, expand Featured Dishes with admin-manageable featured sections. Admin can upload food photos, manage featured sections (create, edit, reorder, delete), and control homepage content through the admin dashboard.

</domain>

<decisions>
## Implementation Decisions

### Photo Upload Workflow
- Both per-item upload AND bulk upload page
- Auto resize + optimize on upload (no manual crop tool)
- JPEG + PNG only (no WebP or other formats)
- 10MB max file size limit
- Drag-drop zone with fallback file picker for bulk upload
- Progress bar per file during upload
- Replace immediately when uploading new photo for item (no history)
- Toast with retry button on upload failure
- Basic validation: check file type, size, dimensions

### Google Drive Integration
- Direct link reference (store Drive URL, serve from Drive, no copy to Supabase)
- Verify image is accessible on save
- Show preview thumbnail after pasting link

### Featured Sections Design
- Predefined sections: Featured Dishes, Most Popular, New Arrivals
- Admin can create additional custom sections (no limit)
- Most Popular: hybrid approach - auto-suggest based on order count, admin can override
- Admin can reorder sections (drag to reorder)
- Admin sets item count per section (not fixed limit)
- Toggle visibility without deleting sections
- Custom icon + accent color per section
- Optional subtitle/description per section
- "View All" links to filtered menu page
- Items can appear in multiple sections
- Search + select to add items to sections
- Drag to reorder items within sections
- No scheduling (sections always visible when enabled)
- Soft delete with 30-day recovery for sections
- Duplicate section button available

### Homepage Layout Changes
- Mixed layout: First section grid, others horizontal scroll
- "See Full Menu" CTA replaces Browse All Dishes
- CTA placed after all sections
- Empty sections auto-hidden on homepage
- Section icons + accent colors shown on homepage
- Navigation arrows always visible on horizontal scroll sections
- Keep existing V8 animation patterns
- Keep existing V8 loading patterns
- Section order controlled by admin only (no personalization)

### Admin Dashboard UI
- Photo upload: both menu item edit page AND dedicated Photos page
- Featured sections: dedicated Sections page in admin
- Side-by-side live preview of homepage changes
- Sidebar menu items for Photos and Sections (separate from each other)
- Draft + Publish workflow (save draft, explicitly publish)
- Basic "Last updated by X at Y" tracking
- Auto-generate alt text from menu item name
- Show photo metadata: dimensions, file size, upload date
- Bulk select + actions available
- Photos page has search + filters (by item name, assigned/unassigned)
- Storage usage statistics displayed

### Supabase Configuration
- Bucket name: `menu-photos`
- Role-based RLS (check if user has 'admin' role)
- 10MB limit enforced in both RLS and client
- Public URLs (no signed URLs)
- Immediate delete from storage (no soft delete for photos)
- Database triggers for cascade cleanup (e.g., delete photo when menu item deleted)
- On-the-fly image transforms using Supabase Storage transform params

### Claude's Discretion
- Bulk upload assignment UX approach
- Grid column count for responsive breakpoints
- Folder structure within storage bucket
- Schema design for featured_sections (normalized tables vs JSON)
- Image caching strategy

</decisions>
