**SaveButton.tsx:**
Per CONTEXT.md locked decision: "Button morphs to checkmark -- subtle scale down, text fades to checkmark icon, brief green pulse, reverts after ~1.5s."

Create a morphing save button component:

```typescript
interface SaveButtonProps {
  onClick: () => Promise<boolean | void>;  // returns success boolean
  disabled?: boolean;
  hasChanges: boolean;
  className?: string;
}
```

States: `idle` | `saving` | `success`

Animation sequence (using Framer Motion):
1. **idle**: Shows Save icon + "Save Changes" text. Uses `Button` component with variant="primary".
2. **saving**: Button scales down slightly (scale: 0.95) via `spring.snappyButton`. Text changes to "Saving..." with `Loader2` spinning icon.
3. **success**: Text fades out, `SuccessCheckmark` (variant="minimal", size={16}) fades in. Button gets brief green background pulse (use CSS `animate-pulse` or FM keyframes over ~1.5s). After 1.5s timeout, reverts to idle.

Use `AnimatePresence` with `mode="wait"` to transition between icon states (Save icon -> Loader -> Checkmark).

Important: This is a **shared** component per CONTEXT.md — "Build reusable save animation component for both admin and customer settings (Phase 51)."

Mark with `"use client"` directive. Use `spring.snappyButton` from `@/lib/motion-tokens` for the scale animation.
