  <name>Task 1: Extract sub-components from 4 largest admin pages</name>
  <files>
    src/app/(admin)/admin/menu/[id]/page.tsx (646 lines)

    src/app/(admin)/admin/categories/page.tsx (577 lines)

  </files>
  <action>
    Admin pages use a DIFFERENT pattern from UI components. Do NOT create subfolders — instead extract sub-components as sibling files in the same route directory.

    **Pattern:**
    ```
    admin/sections/
      page.tsx           # Thin orchestrator (keeps 'use client' + default export)
      SectionList.tsx     # Extracted sub-component
      SectionToolbar.tsx  # Extracted sub-component
    ```

    Next.js App Router ignores non-special filenames (`page`, `layout`, `loading`, `error`, `not-found`, `template`, `route`, `default`). PascalCase component files are safe to co-locate.

    For each page:

    1. Read the file, identify distinct UI sections (forms, lists, toolbars, modals, cards)
    2. Extract each section into a PascalCase sibling file with `'use client'` directive
    3. Pass data via props (move state management to page.tsx or use a shared hook)
    4. page.tsx becomes orchestrator: imports components, manages top-level state, composes layout

    Specific guidance:

    **admin/menu/[id]/page.tsx (646 lines):** Edit form monolith. Extract: MenuItemForm, ImageUploadSection, NutritionSection, or similar logical groupings based on actual content.

    **admin/sections/page.tsx (588 lines):** Reorderable sections UI. Extract: SectionList (drag-and-drop list), SectionEditor (edit panel), SectionToolbar (actions).

    **admin/categories/page.tsx (577 lines):** CRUD. Extract: CategoryList, CategoryForm/Modal, CategoryCard.

    **admin/menu/page.tsx (527 lines):** List + filters. Extract: MenuListTable, MenuFilterBar, or similar.

    Run `pnpm typecheck` after all 4 pages.

  </action>
  <verify>
    `pnpm typecheck` passes with zero errors.
    Each page.tsx is under 400 lines and has a default export.
    Extracted sibling files all have `'use client'` and are under 400 lines.
  </verify>
