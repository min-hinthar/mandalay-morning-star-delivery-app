  <name>Task 1: Upgrade useOfflineSync with sync state, background timer, expiry purge, and drain detection</name>
  <files>src/lib/hooks/useOfflineSync.ts</files>
  <action>
Rewrite `useOfflineSync` hook to add:

**New state:**
- `syncState: 'idle' | 'syncing' | 'synced' | 'error'` -- replaces boolean `isSyncing`. Starts as `'idle'`.
- Keep `isOnline`, `pendingCounts`, `lastSyncResult`

**Expiry purge on mount:**
- In the mount `useEffect`, call `purgeExpiredEntries()` from `@/lib/services/offline-store` (imported from barrel). Log purge count with `console.info`.

**Background timer:**
- When `pendingCounts.total > 0` and `isOnline`, start a 60-second `setInterval` that calls `syncNow()`.
- When `pendingCounts.total === 0` or `!isOnline`, clear the interval.
- Use `useEffect` with `[pendingCounts.total, isOnline]` dependency. Store interval ID in `useRef`. Clean up on unmount.

**Sync state machine:**
- `syncNow()`:
  1. Guard: if `syncState === 'syncing'` or `!navigator.onLine`, return (use `navigator.onLine` for freshness, not React state -- per pitfall #1 in research)
  2. Set `syncState = 'syncing'`
  3. Call `syncPendingItems()`
  4. Set `lastSyncResult`
  5. Call `updatePendingCounts()`
  6. If `pendingCounts.total === 0` after update: set `syncState = 'synced'`, then auto-reset to `'idle'` after 3 seconds via `setTimeout`
  7. If errors exist: set `syncState = 'error'`, auto-reset to `'idle'` after 5 seconds
  8. Otherwise: set `syncState = 'idle'`

**Drain detection:**
- Add `onDrain?: () => void` callback parameter to the hook (optional)
- When queue transitions from non-empty to empty after sync, call `onDrain?.()`
- This enables consumers (like StopDetailView) to trigger `router.refresh()` after sync completes

**Return shape update:**
- Replace `isSyncing: boolean` with `syncState: 'idle' | 'syncing' | 'synced' | 'error'`
- Add backward compat: keep `isSyncing` as `syncState === 'syncing'` computed property in return
- Everything else stays the same

- Get `{ isOnline, queueStatusUpdate }` from `useOfflineSync()`
- Modify `updateStatus()`:
  1. If `navigator.onLine` (use navigator directly for freshness):
     - Try the existing `fetch()` call
     - On network `TypeError` catch: fall through to queue
     - On 4xx: show error (existing behavior)
     - On 5xx: fall through to queue
  2. If offline OR fell through from network/5xx error:

**ExceptionModal.tsx:**

- Modify `handleSubmit()`:

     - Note: The full exception report will fail to sync because the exception endpoint expects POST not PATCH. The simplification here is: queue a "skipped" status update, which uses the existing stop PATCH endpoint.      - Call `onClose()` to dismiss modal

     - Set `hasPhoto` to true (optimistic)
