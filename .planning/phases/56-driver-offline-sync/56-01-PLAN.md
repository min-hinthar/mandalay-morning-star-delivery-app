**retry.ts (NEW FILE):**

Create `src/lib/services/offline-store/retry.ts` with `"use client"` directive:

```typescript
export function getBackoffDelay(attempt: number, baseMs = 2000, maxMs = 32000): number {
  return Math.min(baseMs * Math.pow(2, attempt), maxMs);
}

function sleep(ms: number): Promise<void> {
  return new Promise(resolve => setTimeout(resolve, ms));
}
```

Export `retryWithBackoff` async function:
- Parameters: `url: string`, `options: RequestInit`, `idempotencyKey: string`, `maxAttempts = 5`
- Returns: `Promise<{ ok: boolean; status: number; permanentFailure: boolean }>`
- Logic:
  - Loop `maxAttempts` times
  - Each attempt: `fetch(url, { ...options, headers: { ...options.headers, 'Idempotency-Key': idempotencyKey } })`
  - If `response.ok` (2xx): return `{ ok: true, status: response.status, permanentFailure: false }`
  - If `response.status >= 400 && response.status < 500`: return `{ ok: false, status: response.status, permanentFailure: true }` (no retry)
  - If `response.status >= 500`: log `console.warn('Retry attempt ${attempt + 1}/${maxAttempts} for ${url}: ${response.status}')`, continue to backoff
  - On catch (network TypeError): log similarly, continue to backoff
  - Between attempts (except after last): `await sleep(getBackoffDelay(attempt))` -- NO jitter per user decision
  - After all attempts exhausted: return `{ ok: false, status: 0, permanentFailure: false }`

Also export a `purgeExpiredEntries` async function:
- Parameters: `maxAgeMs = 2 * 60 * 60 * 1000` (2 hours per locked decision)
- Reads all items from `pendingStatus.getAll()`, `pendingPhotos.getAll()`, `pendingLocations.getAll()`
- For each item: if `Date.now() - new Date(item.createdAt).getTime() > maxAgeMs`, remove it
- Returns `number` of purged items
- Import `pendingStatus, pendingPhotos, pendingLocations` from `./stores`

**sync.ts:**
