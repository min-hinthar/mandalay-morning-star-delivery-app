**Create `src/app/api/health/route.ts`:**

Export a `GET` handler:

```typescript
import { NextRequest, NextResponse } from "next/server";
import { checkEnvVars, runDeepChecks, type HealthResponse, type ServiceStatus } from "@/lib/health";
```

**Handler logic:**

1. Parse `?deep=true` from `request.nextUrl.searchParams`
2. Always run `checkEnvVars()` -- returns `EnvCheckResult`
3. Build config-only service status for each service:
   - Check if relevant env vars are present (Supabase: `NEXT_PUBLIC_SUPABASE_URL` + `NEXT_PUBLIC_SUPABASE_ANON_KEY`; Stripe: `STRIPE_SECRET_KEY`; Resend: `RESEND_API_KEY`)
   - Set `configured: true/false` and `status: configured ? "healthy" : "down"` for config-only mode
4. Build config-only route status: `{ path, reachable: true }` (can't verify without fetch)

5. If `deep === true`:
   - Call `runDeepChecks(request.nextUrl.origin)` which returns full service statuses + route statuses
   - Merge deep results into the response (override config-only with deep results)

6. Compute `production_ready`:
   - `envResult.all_critical_present` is true AND
   - All service statuses are `"healthy"` AND
   - All routes are reachable
   - In config-only mode, `production_ready` requires all critical env vars present (services assumed healthy if configured)

7. Compute top-level `status`: worst of all service statuses (healthy < degraded < down)

8. Build `HealthResponse` with:
   - `timestamp`: `new Date().toISOString()`
   - `version`: `process.env.VERCEL_GIT_COMMIT_SHA?.slice(0, 7) ?? "dev"`
   - `environment`: `process.env.VERCEL_ENV ?? "development"`
   - `services`, `routes`, `env` from above

9. Return `NextResponse.json(response, { status: allHealthy ? 200 : 503 })`
   - 200 if top-level status is "healthy"
   - 503 if "degraded" or "down"

10. Set `Cache-Control: no-store, no-cache, must-revalidate` header on the response.

**Update `next.config.ts`:**

In the `async headers()` function, add a new entry BEFORE the existing font/icon headers:

```typescript
{
  source: "/api/health",
  headers: [
    { key: "Access-Control-Allow-Origin", value: "*" },
    { key: "Access-Control-Allow-Methods", value: "GET, OPTIONS" },
    { key: "Access-Control-Allow-Headers", value: "Content-Type" },
  ],
},
```

Per locked decision: CORS allows all origins. Per research: simpler to set via next.config.ts headers() than in route handler.

Do NOT add Cache-Control in next.config.ts -- set it in the route handler response so it applies to both the JSON response and any preflight.
