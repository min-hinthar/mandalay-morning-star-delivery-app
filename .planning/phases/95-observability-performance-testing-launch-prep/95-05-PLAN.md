  <action>
    Create `scripts/load-test.js` -- k6 load test script for 50 concurrent checkout submissions.

    1. **k6 script structure:**
       ```javascript
       import http from 'k6/http';
       import { check, sleep } from 'k6';
       import { SharedArray } from 'k6/data';

       export const options = {
         scenarios: {
           checkout_surge: {
             executor: 'constant-vus',
             vus: 50,
             duration: '2m',
           },
         },
         thresholds: {
           http_req_duration: ['p(95)<3000'],  // p95 < 3 seconds
           http_req_failed: ['rate<0.01'],      // <1% error rate
         },
       };
       ```

    2. **Setup function** (runs once before test):
       - Authenticate a test user via Supabase auth API to get JWT
       - Fetch menu items from `/api/menu` to get valid item IDs
       - Store auth token and item IDs for VUs to use
       ```javascript
       export function setup() {
         // Authenticate test user
         const authRes = http.post(`${__ENV.SUPABASE_URL}/auth/v1/token?grant_type=password`, JSON.stringify({
           email: __ENV.TEST_USER_EMAIL,
           password: __ENV.TEST_USER_PASSWORD,
         }), {
           headers: { 'Content-Type': 'application/json', 'apikey': __ENV.SUPABASE_ANON_KEY },
         });

         const token = JSON.parse(authRes.body).access_token;

         // Fetch menu items
         const menuRes = http.get(`${__ENV.BASE_URL}/api/menu`);
         const items = JSON.parse(menuRes.body).data?.items || [];

         return { token, items: items.slice(0, 5).map(i => i.id) };
       }
       ```

    3. **Default function** (runs per VU iteration):
       - Build checkout payload with random item selection from setup data
       - POST to `/api/checkout/session` with auth header
       - Check response status and body structure
       - Sleep 1s between iterations (realistic pacing)

    4. **Environment variables** (passed via `k6 run -e`):
       - `BASE_URL` -- app URL (default: http://localhost:3000)
       - `SUPABASE_URL` -- Supabase project URL
       - `SUPABASE_ANON_KEY` -- Supabase anon key for auth
       - `TEST_USER_EMAIL` -- test user email
       - `TEST_USER_PASSWORD` -- test user password

    5. **Run command** (document in script header comment):
       ```bash
       k6 run scripts/load-test.js \
         -e BASE_URL=http://localhost:3000 \
         -e SUPABASE_URL=... \
         -e SUPABASE_ANON_KEY=... \
         -e TEST_USER_EMAIL=test@example.com \
         -e TEST_USER_PASSWORD=testpass123
       ```

    6. **Pass/fail criteria** (built into k6 thresholds):
       - p95 response time < 3 seconds
       - HTTP error rate < 1%
       - k6 exits with code 0 if thresholds pass, non-zero if any fail

    NOTE: k6 must be installed separately (`choco install k6` or `winget install grafana.k6`). The script is not an npm dependency. Add a comment at the top of the file noting the installation requirement.

    NOTE: Each VU request will likely fail at the Stripe session creation step (duplicate order per user per Saturday). This is expected -- the test validates API throughput and validation, not payment completion.
