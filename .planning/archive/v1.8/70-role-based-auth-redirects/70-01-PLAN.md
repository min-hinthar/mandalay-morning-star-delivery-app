**Create `src/lib/auth/role-redirect.ts`:**
- Export `RoleRedirectResult` type: `{ path: string; role: string; driverStatus?: "active" | "inactive" | "no_record" }`
- Export `async function getRoleDashboard(supabase: SupabaseClient, userId: string): Promise<RoleRedirectResult>`
- Query `profiles.role` for the user
- If no profile row exists (edge case): auto-create profile with `role='customer'` using service client pattern. Get user email from `supabase.auth.getUser()`. Insert `{ id: userId, email, role: 'customer' }`. Return `{ path: "/menu", role: "customer" }`. (Self-healing per locked decision)
- If role is `admin`: return `{ path: "/admin", role: "admin" }`
- If role is `driver`: query `drivers` table for `user_id` WITHOUT `is_active` filter (need to distinguish deactivated vs no-record):
  - No driver record → `{ path: "/driver/onboard", role: "driver", driverStatus: "no_record" }`
  - `is_active === false` → `{ path: "/driver/deactivated", role: "driver", driverStatus: "inactive" }`
  - `is_active === true` → `{ path: "/driver", role: "driver", driverStatus: "active" }`
- Default (customer or any other role): return `{ path: "/menu", role: "customer" }`
- Wrap entire function in try/catch — on DB error, log to console.error and return `{ path: "/", role: "unknown" }` as safe fallback
