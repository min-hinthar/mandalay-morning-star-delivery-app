## What Was Built

### 1. Order Audit Log Migration (011_order_audit_log.sql)

- `order_audit_log` table with action types: status_change, cancel, refund, edit
- Actor tracking (actor_id, actor_role: customer/admin/driver/system)
- JSONB old_value/new_value for complete change history
- RLS: admins see all, customers see own orders
- Immutable: no UPDATE or DELETE policies
- Added `refunded_quantity` column to `order_items` for partial refund tracking

### 2. Order Validation Schemas (src/lib/validations/order.ts)

- `cancelOrderSchema`: reason (5-500 chars), notifyCustomer flag
- `refundOrderSchema`: item-level with orderItemId, quantity, reason
- `changeOrderStatusSchema`: status enum with optional reason

### 3. Cancel API (POST /api/admin/orders/[id]/cancel)

- Admin auth required
- Updates order status to 'cancelled'
- Creates audit log entry with old/new status
- Blocks cancellation of delivered orders
- Customer notification flag (TODO: integrate notification service)

### 4. Refund API (POST /api/admin/orders/[id]/refund)

- Admin auth required
- Item-level refunds with quantity validation
- Calculates refund amount proportionally
- Updates `refunded_quantity` on order_items
- Optional shipping refund
- Audit log with complete old/new item states
