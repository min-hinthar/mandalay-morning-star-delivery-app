  <name>Task 1: Refactor RouteError to CSS-only with enhanced UX</name>
  <files>src/components/ui/RouteError.tsx</files>
  <action>
Rewrite `RouteError.tsx` to remove ALL framer-motion imports and use CSS-only animation. Specific changes:

1. **Remove framer-motion:** Delete `import { m } from "framer-motion"`. Replace `m.div` elements with plain `div` elements.

2. **CSS animation:** Apply `animate-fade-in-up` class (from `src/styles/animations.css`) to the outer container div. This provides the same fade-in + slide-up effect as the current framer-motion animation.

3. **Add retry counter:** Use `useRef(0)` for retry count (survives re-renders without resetting). Use `useState(false)` for `showHomeEmphasis`. On each retry click, increment ref. When `retryCount.current >= 2`, set `showHomeEmphasis` to true.

4. **Button hierarchy:**
   - Default state: retry = `variant="default"` (primary), go home = `variant="ghost"` (secondary)
   - After 2+ retries: retry = `variant="outline"` (demoted), go home = `variant="default"` (promoted)
   - Per user decision: retry is primary, go home is secondary/ghost

5. **Morning Star branding:**
   - Add `import Image from "next/image"`
   - Before the alert icon, render: `<Image src="/logo.png" alt="Morning Star" width={48} height={48} className="mx-auto mb-4" style={{ height: 'auto' }} />`
   - Alert icon container: use `bg-status-error-bg` background and `text-status-error` icon color (NOT `brand-red` -- it's a ghost token)

6. **Friendly personality:**
   - Heading: `"Oops, we hit a bump!"` (not "Something went wrong")
   - Body with context: `"We couldn't load the ${context}. Give it another shot!"`
   - Body without context: `"Something unexpected happened. Give it another shot!"`

7. **Dev error details (adding missing stack trace):** Current code only shows `error.message` in dev mode. Enhance to also show `error.stack` (currently missing). Use a `pre` tag with `bg-surface-tertiary p-3 rounded-md overflow-auto max-h-24 mb-6 text-left text-xs` classes. Format: `{error.message}{error.stack && '\n\n' + error.stack}`. This adds stack trace visibility that the current implementation lacks.

8. **Sentry logging:** Keep existing `captureException` call. Add `console.error(error)` as dev fallback (Sentry is no-op in dev).

9. **Layout:** Change outer container from `min-h-screen` to `min-h-[60vh]` so the app shell (nav/sidebar) remains visible. Keep `flex items-center justify-center p-4`. Use `bg-background` class.

10. **Semantic tokens:** Replace `text-charcoal` with `text-text-primary`. Replace `text-muted-foreground` with `text-text-secondary`. These are the proper design system tokens.

Do NOT import framer-motion anywhere in this file. The `'use client'` directive must remain.
</action>
