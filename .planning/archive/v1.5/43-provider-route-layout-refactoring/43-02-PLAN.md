  <action>
  1. Create `src/lib/hooks/useNavigationGuard.ts`:
     - "use client" directive
     - Interface `UseNavigationGuardOptions`:
       - `enabled: boolean` -- guard active when true (typically: cart has items)
       - `allowedPaths?: string[]` -- routes that should NOT trigger the guard (e.g., ["/cart", "/checkout", "/menu", "/"])
     - Hook returns: `{ showModal: boolean; proceed: () => void; cancel: () => void }`
     - Implementation:
       a. **beforeunload** -- when `enabled`, add `beforeunload` listener that calls `e.preventDefault()`. This shows native browser "Leave page?" dialog on tab close/external nav. Clean up on unmount or when `enabled` changes.
       b. **popstate (back/forward button)** -- when `enabled`, listen for `popstate`. On trigger: push current state back via `window.history.pushState(null, "", pathname)` to prevent leaving, then `setShowModal(true)`. Clean up on unmount.
       c. **proceed()** -- sets `showModal` to false, then navigates back via `window.history.back()` (the user confirmed they want to leave).
       d. **cancel()** -- sets `showModal` to false, clears pending navigation.
     - NOTE on `onNavigate` Link prop: This is a Next.js 16 Link prop. The hook itself does NOT handle Link navigation. Instead, the checkout/cart pages should use `<Link onNavigate={(e) => { if (guardEnabled) { e.preventDefault(); setShowModal(true); pendingHref = href; } }}>` pattern inline where needed. The hook focuses on browser-level events (back button, tab close) which cannot be handled by Link props.
     - Import and use `usePathname` from "next/navigation"
     - Import `useState, useEffect, useCallback, useRef` from "react"

2. Create `src/components/ui/cart/CartNavigationGuard.tsx`:

     - `onLeave: () => void` -- user confirmed they want to leave
     - `variant: "checkout" | "cart"` -- determines copy
